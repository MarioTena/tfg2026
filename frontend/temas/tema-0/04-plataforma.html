Cómo está organizada

La plataforma combina teoría breve, bloques de práctica, mini-retos, ejercicios y un playground para probar código.

Qué encontrarás

  • rutas por temas
  • pantallas de concepto
  • mini-retos para consolidar
  • ejercicios para resolver
  • editor y ejecución de código

Idea clave

La plataforma no está pensada para solo leer. Está pensada para alternar entender, probar y corregir.

Cómo avanzar bien

1. Entiende el concepto

Lee la idea principal y qué problema resuelve antes de ir a la práctica.

2. Haz mini-retos

Los mini-retos sirven para conectar conceptos y comprobar si realmente los manejas.

3. Resuelve ejercicios

Aquí ya no se trata de mirar ejemplos, sino de construir tu propia solución.

4. Usa el playground

Es el lugar para ejecutar código, probar variantes y aprender del feedback.

Qué no deberías hacer

Hay hábitos que frenan mucho el aprendizaje al principio.

Evita esto

  • saltar entre temas sin base
  • copiar soluciones sin pensar
  • ignorar los errores
  • usar el editor solo para ejecutar, no para experimentar

Haz esto en cambio

Lee menos tiempo seguido y prueba más. Aunque avances más despacio, aprenderás mejor.

Qué papel tiene el playground

El playground no es solo un editor. Es la zona donde conviertes una idea en una prueba real. Ahí es donde se ve si has entendido un concepto o solo lo has leído.

Siguiente paso

Ya sabes cómo está organizada la plataforma. Ahora toca aprender a ver los errores como parte normal del proceso de programar.
