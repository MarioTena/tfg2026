6.1 · Qué es un diccionario: guardar información por clave y valor

¿Qué problema resuelven los diccionarios?

Las listas son muy útiles cuando el orden importa y trabajas con posiciones. Pero a veces no quieres acceder por índice, sino por significado.

Por ejemplo, si tienes información de una persona, esto no es muy expresivo: 

persona = ["Ana", 30, "Madrid"]

¿Qué representa cada posición? Hay que recordarlo. En cambio, con un diccionario puedes nombrar cada dato.

Un diccionario guarda pares clave → valor. Permite acceder a la información por nombre, no por posición.

¿Qué es un diccionario?

Un diccionario es una estructura de datos que almacena valores asociados a claves. Se escribe entre llaves {}. 

persona = {
    "nombre": "Ana",
    "edad": 30,
    "ciudad": "Madrid"
}

Aquí las claves son "nombre", "edad" y "ciudad".

Ventajas frente a una lista

  • el código es más legible
  • cada dato tiene significado explícito
  • es ideal para representar entidades reales
  • evita depender de recordar posiciones numéricas

Ejemplos reales

  • una persona: nombre, edad, email
  • un producto: nombre, precio, stock
  • una película: título, año, director
  • una configuración: idioma, tema, notificaciones

Crear un diccionario vacío

configuracion = {}

Igual que con listas, a veces empiezas vacío y luego añades datos.

Claves únicas

En un diccionario no debería haber claves repetidas. Cada clave identifica un dato concreto. 

producto = {
    "nombre": "Teclado",
    "precio": 40
}

Qué debes entender desde el principio

Una lista responde a “¿qué hay en la posición 0, 1 o 2?”. Un diccionario responde a “¿qué valor tiene la clave nombre, edad o precio?”.

💭 Piensa antes de ejecutar

coche = {
    "marca": "Toyota",
    "modelo": "Corolla",
    "anio": 2022
}

print(coche)
  • ¿qué estructura se imprime?
  • ¿qué diferencia ves con una lista?

Errores conceptuales frecuentes

  • confundir diccionarios con listas
  • usar corchetes en vez de llaves al crearlos
  • no entender la diferencia entre clave y valor
  • pensar que se accede por posición como en una lista

Mini-reto guiado

Crea un diccionario que represente un libro con las claves titulo, autor y anio. 

libro = {
    "titulo": "Clean Code",
    "autor": "Robert C. Martin",
    "anio": 2008
}

print(libro)

Pregunta de entrevista

¿Qué diferencia conceptual hay entre una lista y un diccionario?

Respuesta sólida: una lista organiza datos por posición; un diccionario organiza datos por claves con significado.
