Qué vas a entender hoy

Un diccionario permite guardar información por nombre, no por posición. Cada dato tiene una clave y un valor asociado.

Objetivo Clave → valor

Qué problema resuelve

Las listas son muy útiles cuando importa el orden, pero a veces no quieres acceder por índice, sino por significado. Un diccionario te permite nombrar cada dato de forma explícita.

Persona

Guardar nombre, edad y ciudad con significado claro.

Producto

Representar nombre, precio y stock.

Configuración

Guardar idioma, tema o notificaciones.

Lectura clara

Evitar recordar qué significa cada posición de una lista.

Idea clave: un diccionario organiza datos por claves con significado, no por posiciones.

Qué debes vigilar

  • un diccionario usa llaves {}
  • cada dato se guarda como clave: valor
  • las claves sirven para identificar cada valor
  • no debes pensar en índices como en una lista
Piensa así:
una lista responde a “¿qué hay en la posición 0?” y un diccionario responde a “¿qué valor tiene la clave nombre?”.

Qué es un diccionario

Un diccionario es una estructura de datos que almacena valores asociados a claves. Se escribe entre llaves {}. 

persona = {
    "nombre": "Ana",
    "edad": 30,
    "ciudad": "Madrid"
}

Aquí las claves son "nombre", "edad" y "ciudad".

Dónde aparecen los diccionarios en problemas reales

Usuarios

Guardar datos personales o de acceso.

Productos

Representar precio, stock, nombre o categoría.

Películas o libros

Asociar título, año, autor o director.

Configuraciones

Gestionar opciones del sistema con nombres claros.

Cuando los datos tienen nombre propio y quieres acceder a ellos por significado, el diccionario suele ser la mejor opción.

Ventajas frente a una lista

Más legible

Cada dato tiene una clave clara.

Más expresivo

No tienes que recordar qué representa cada posición.

Más realista

Encaja muy bien para representar entidades reales.

Más mantenible

El código suele entenderse mejor con el paso del tiempo.

Crear un diccionario vacío

configuracion = {}

Igual que con listas, a veces empiezas vacío y luego añades datos.

Claves únicas

En un diccionario no debería haber claves repetidas. Cada clave identifica un dato concreto. 

producto = {
    "nombre": "Teclado",
    "precio": 40
}

Qué debes entender desde el principio

Lista

Responde a “¿qué hay en la posición 0, 1 o 2?”.

Diccionario

Responde a “¿qué valor tiene la clave nombre, edad o precio?”.

Posición

En listas importa mucho.

Significado

En diccionarios importa más que la posición.

Piensa antes de ejecutar

Antes de probar el código, piensa qué estructura representa y cómo se diferencia de una lista.

Código

coche = {
    "marca": "Toyota",
    "modelo": "Corolla",
    "anio": 2022
}

print(coche)

Preguntas

  • ¿qué estructura se imprime?
  • ¿qué diferencia ves con una lista?
Pista: aquí cada dato tiene una clave con significado.

Mini-reto guiado

Ahora toca representar una entidad real con claves claras.

Reto Muy fácil

Tu misión

Crea un diccionario que represente un libro con las claves titulo, autor y anio.

Resolver en playground

Pista

Piensa en el libro como una entidad con varios datos nombrados, no como una secuencia de posiciones.

Mostrar solución 
libro = {
    "titulo": "Clean Code",
    "autor": "Robert C. Martin",
    "anio": 2008
}

print(libro)

Errores frecuentes

Error 1

Confundir diccionarios con listas.

Error 2

Usar corchetes en vez de llaves al crearlos.

Error 3

No entender la diferencia entre clave y valor.

Error 4

Pensar que se accede por posición como en una lista.

Buenas prácticas

Claves claras

Usa nombres que expliquen bien cada dato.

Entidad coherente

No mezcles en un mismo diccionario datos sin relación.

Criterio uniforme

Mantén el mismo estilo para nombrar claves.

Piensa en significado

Usa diccionario cuando el nombre del dato importa más que su orden.
