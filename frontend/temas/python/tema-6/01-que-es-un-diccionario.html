Qué vas a entender hoy

Un diccionario permite guardar información por nombre, no por posición. Cada dato tiene una clave y un valor asociado.

Objetivo Clave → valor

Qué problema resuelve

Las listas son muy útiles cuando importa el orden, pero a veces no quieres acceder por índice, sino por significado. Un diccionario te permite nombrar cada dato de forma explícita.

Persona

Guardar nombre, edad y ciudad con significado claro.

Producto

Representar nombre, precio y stock.

Configuración

Guardar idioma, tema o notificaciones.

Lectura clara

Evitar recordar qué significa cada posición de una lista.

Idea clave: un diccionario organiza datos por claves con significado, no por posiciones.

Qué debes vigilar

  • un diccionario usa llaves {}
  • cada dato se guarda como clave: valor
  • las claves sirven para identificar cada valor
  • no debes pensar en índices como en una lista
Piensa así: una lista responde a “¿qué hay en la posición 0?” y un diccionario responde a “¿qué valor tiene la clave nombre?”.

Qué es un diccionario

Un diccionario es una estructura de datos que almacena valores asociados a claves. Se escribe entre llaves {}. 

persona = {
    "nombre": "Ana",
    "edad": 30,
    "ciudad": "Madrid"
}

Aquí las claves son "nombre", "edad" y "ciudad".

Dónde aparecen los diccionarios en problemas reales

Usuarios

Guardar datos personales o de acceso.

Productos

Representar precio, stock, nombre o categoría.

Películas o libros

Asociar título, año, autor o director.

Configuraciones

Gestionar opciones del sistema con nombres claros.

Cuando los datos tienen nombre propio y quieres acceder a ellos por significado, el diccionario suele ser la mejor opción.

Lista frente a diccionario

Lista

Sirve cuando importa el orden o la posición de cada elemento.

Diccionario

Sirve cuando importa el nombre o el significado de cada dato.

Lista pregunta

¿Qué hay en la posición 0 o 1?

Diccionario pregunta

¿Qué valor tiene la clave precio o edad?

Ventajas frente a una lista

Más legible

Cada dato tiene una clave clara.

Más expresivo

No tienes que recordar qué representa cada posición.

Más realista

Encaja muy bien para representar entidades reales.

Más mantenible

El código suele entenderse mejor con el paso del tiempo.

Entidad y atributos

Una forma útil de pensar un diccionario es esta: representa una entidad y cada clave es uno de sus atributos.

Entidad

Un usuario, un producto, una película o una configuración.

Atributos

nombre, edad, precio, stock, activo.

Idea práctica

Cada clave describe qué representa ese valor.

Ventaja

El programa se lee con más sentido que usando posiciones numéricas.

Ejemplos reales

Persona

nombre, edad, email

Producto

nombre, precio, stock

Película

titulo, anio, director

Configuración

idioma, tema, notificaciones

Crear un diccionario vacío

configuracion = {}

Igual que con listas, a veces empiezas vacío y luego añades datos.

Casos simples importantes

Vacío

{}

Una sola clave

{"nombre": "Ana"}

Varios tipos

{"nombre": "Ana", "edad": 30, "activo": True}

Idea útil

Los valores pueden ser de tipos distintos si tiene sentido para el problema.

Claves únicas

En un diccionario no debería haber claves repetidas. Cada clave identifica un dato concreto. 

producto = {
    "nombre": "Teclado",
    "precio": 40
}
La clave no es un adorno. Es el nombre exacto con el que luego pedirás ese dato.

Cómo pensar si te conviene un diccionario

Paso 1

Pregúntate si los datos tienen nombres naturales.

Paso 2

Si vas a pedir información como precio o stock, suele encajar un diccionario.

Paso 3

Si solo te importa una secuencia ordenada, quizá una lista sea mejor.

Paso 4

Decide qué claves representan mejor la realidad del problema.
