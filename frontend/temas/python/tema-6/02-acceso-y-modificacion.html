Qué vas a entender hoy

Un diccionario no solo sirve para guardar datos: también tienes que saber acceder a ellos, cambiarlos y añadir información nueva.

Objetivo Acceso y cambio

Qué problema resuelve

Crear un diccionario es solo el primer paso. En programación real necesitas consultar un dato concreto, actualizarlo cuando cambia o añadir nuevas claves cuando aparece más información.

Leer datos

Acceder a un valor usando su clave.

Actualizar

Cambiar el valor de una clave que ya existe.

Añadir

Crear una nueva clave con su valor asociado.

Seguridad

Evitar errores cuando una clave no existe.

Idea clave: trabajar con diccionarios significa moverte por claves, no por posiciones.

Qué debes vigilar

  • escribir bien cada clave
  • distinguir entre actualizar y añadir
  • tener cuidado con claves que no existen
  • usar get() cuando necesites más seguridad
Piensa así:
acceder a diccionario["clave"] es pedir un dato concreto por su nombre.

Acceder a un valor por su clave

Para acceder a un valor en un diccionario, usas la clave entre corchetes. 

persona = {
    "nombre": "Ana",
    "edad": 30,
    "ciudad": "Madrid"
}

print(persona["nombre"])
print(persona["edad"])

Esto devuelve los valores asociados a esas claves.

Modificar un valor existente

persona = {
    "nombre": "Ana",
    "edad": 30
}

persona["edad"] = 31
print(persona)

Si la clave ya existe, su valor se actualiza.

Añadir una nueva clave

persona = {
    "nombre": "Ana",
    "edad": 30
}

persona["email"] = "ana@email.com"
print(persona)

Si la clave no existía, se crea.

Dos ideas que debes distinguir

Actualizar

La clave ya existía y cambias su valor.

Añadir

La clave no existía y pasa a formar parte del diccionario.

Misma sintaxis

En ambos casos usas diccionario[clave] = valor.

Diferencia real

Lo que cambia es si esa clave ya estaba o no.

Error típico: acceder a una clave inexistente

producto = {
    "nombre": "Ratón",
    "precio": 25
}

print(producto["stock"])

Esto da error porque la clave "stock" no existe.

Usar get() para acceder con más seguridad

producto = {
    "nombre": "Ratón",
    "precio": 25
}

print(producto.get("stock"))
print(producto.get("stock", 0))

get() devuelve None o un valor por defecto si la clave no existe. Es muy útil para evitar errores.

Cuando no estás seguro de si una clave existe, get() suele ser mejor que acceder directamente con corchetes.

Comprobar si una clave existe

usuario = {
    "nombre": "Luis",
    "activo": True
}

print("activo" in usuario)
print("email" in usuario)

El operador in permite comprobar si una clave está en el diccionario.

Eliminar una clave

datos = {
    "nombre": "Marta",
    "edad": 28,
    "ciudad": "Sevilla"
}

del datos["ciudad"]
print(datos)

del elimina la clave y su valor asociado.

Cómo pensar acceso y modificación antes de escribirlo

Paso 1

Identifica qué clave representa el dato que necesitas.

Paso 2

Decide si vas a leer, actualizar, añadir o eliminar.

Paso 3

Piensa si la clave puede no existir.

Paso 4

Si hay duda, usa get() o comprueba con in.

Piensa antes de ejecutar

Antes de lanzar el código, sigue mentalmente qué claves cambian y cuáles siguen igual.

Código

alumno = {
    "nombre": "Carlos",
    "nota": 8
}

alumno["nota"] = 9
alumno["curso"] = "Python"
print(alumno)

Errores frecuentes

Error 1

Escribir mal una clave.

Error 2

Acceder a una clave que no existe.

Error 3

Olvidar que las claves de texto deben ir entre comillas.

Error 4

Confundir actualizar una clave con añadir una nueva.

Buenas prácticas

Claves consistentes

Mantén siempre el mismo criterio de nombres.

Comprueba antes

Si no estás seguro de una clave, usa get() o in.

Piensa en intención

Diferencia leer, actualizar, crear o eliminar.

Evita suposiciones

No des por hecho que todos los diccionarios tienen siempre las mismas claves.
