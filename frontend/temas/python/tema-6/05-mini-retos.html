Diccionarios Claves items()

Objetivo

Vas a practicar cómo representar entidades, actualizar datos y recorrer información con significado.

Idea clave: con diccionarios trabajas con significado, no con posiciones.

Cómo usar esta pantalla

  • lee el reto
  • identifica qué representa cada clave
  • decide si necesitas una clave concreta o recorrer todo
  • pruébalo en el playground y cambia datos
La pregunta importante aquí no es “cómo se escribe”, sino “qué datos tiene sentido guardar con nombre”.

Misiones rápidas

Mini-reto 1 · Ficha de usuario

Recorre un diccionario con claves y valores.

usuario = {
    "nombre": "Ana",
    "edad": 30,
    "activo": True
}

for clave, valor in usuario.items():
    print(clave, ":", valor)

Mini-reto 2 · Actualizar precio

Modifica un valor existente usando su clave.

producto = {
    "nombre": "Teclado",
    "precio": 50
}

producto["precio"] = 45
print(producto)

Mini-reto 3 · Comprobar clave

Verifica si una clave existe antes de usarla.

config = {
    "tema": "oscuro",
    "idioma": "es"
}

if "tema" in config:
    print("La clave existe")

Mini-reto 4 · Valor por defecto

Usa get() cuando una clave puede no existir.

usuario = {
    "nombre": "Luis"
}

print(usuario.get("email", "No informado"))

Mini-reto 5 · Recorrer solo valores

Accede únicamente a los valores del diccionario.

medidas = {
    "alto": 180,
    "ancho": 90,
    "profundo": 50
}

for valor in medidas.values():
    print(valor)

Mini-reto 6 · Eliminar una clave

Quita información que ya no necesitas.

alumno = {
    "nombre": "Marta",
    "nota": 8,
    "curso": "Python"
}

alumno.pop("curso")
print(alumno)

Cómo pensar este tipo de ejercicios

Antes de crear el diccionario, piensa en la entidad y sus atributos.

Paso a paso

  1. decide qué representa el diccionario
  2. piensa qué claves son necesarias
  3. usa items() si necesitas clave y valor
  4. usa get() si hay claves opcionales
  5. evita mezclar información sin estructura

Regla útil

Si cada dato tiene una etiqueta natural como nombre, precio, edad o estado, probablemente un diccionario encaja bien.

Pregunta típica de entrevista

Pregunta: ¿Qué ventaja tiene un diccionario para representar una entidad como un producto o un usuario?

Respuesta sólida: permite asociar cada dato a una clave con significado, haciendo el código más legible y fácil de mantener.
