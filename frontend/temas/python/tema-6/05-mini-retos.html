Tema 6 · Python

6.5 · Mini retos: integrar diccionarios, bucles y decisiones

Objetivo de esta parte

Ahora toca combinar lo aprendido para usar diccionarios en problemas más realistas.

Mini-reto 1 · Ficha de usuario

usuario = {
    "nombre": "Ana",
    "edad": 30,
    "activo": True
}

for clave, valor in usuario.items():
    print(clave, ":", valor)

Mini-reto 2 · Actualizar precio

producto = {
    "nombre": "Teclado",
    "precio": 50
}

producto["precio"] = 45
print(producto)

Mini-reto 3 · Comprobar clave

config = {
    "tema": "oscuro",
    "idioma": "es"
}

if "tema" in config:
    print("La clave existe")

Mini-reto 4 · Valor por defecto

usuario = {
    "nombre": "Luis"
}

print(usuario.get("email", "No informado"))

Mini-reto 5 · Recorrer solo valores

medidas = {
    "alto": 180,
    "ancho": 90,
    "profundo": 50
}

for valor in medidas.values():
    print(valor)

Mini-reto 6 · Eliminar una clave

alumno = {
    "nombre": "Marta",
    "nota": 8,
    "curso": "Python"
}

alumno.pop("curso")
print(alumno)

Cómo pensar este tipo de ejercicios

  1. decide qué representa el diccionario
  2. piensa qué claves son necesarias
  3. usa items() si necesitas clave y valor
  4. usa get() si hay claves opcionales
  5. evita mezclar información sin estructura

Pregunta de entrevista

¿Qué ventaja tiene un diccionario para representar una entidad como un producto o un usuario?

Respuesta sólida: permite asociar cada dato a una clave con significado, haciendo el código más legible y fácil de mantener.
