Qué entrenas aquí

En este bloque no solo escribes diccionarios. Aprendes a leerlos, razonar qué estructura conviene y detectar errores frecuentes con claves.

Lectura Depuración Práctica guiada

Objetivo

Estos retos te ayudan a dejar de ver el diccionario como una simple sintaxis con llaves y empezar a verlo como una estructura con intención.

Idea clave: un diccionario correcto no solo se escribe bien. También representa bien el problema.

Cómo usar esta pantalla

  • lee el código antes de ejecutarlo
  • identifica qué representa cada clave
  • piensa si una clave existe o no
  • decide si conviene lista o diccionario
Si no puedes explicar qué significa cada clave o por qué esa estructura es adecuada, todavía no has entendido del todo el diccionario.

Reto 1 · ¿Qué representa mejor este caso?

Situación

Quieres guardar el nombre, precio y stock de un producto.

Qué debes pensar

¿Tiene más sentido una lista o un diccionario? ¿Por qué conviene acceder por clave y no por posición?

Reto 2 · Acceso por clave

Código

pelicula = {
    "titulo": "Inception",
    "anio": 2010,
    "director": "Christopher Nolan"
}

print(pelicula["anio"])

Pregunta

¿Qué imprime exactamente? ¿Qué representa esa clave dentro del diccionario?

Reto 3 · Detecta el error

Código incorrecto

usuario = {
    "nombre": "Ana",
    "activo": True
}

print(usuario["email"])

Pregunta

¿Por qué falla? ¿Qué alternativa sería más segura si no sabes si esa clave existe?

No basta con decir “da error”. Debes identificar por qué falla y qué método encajaría mejor.

Reto 4 · Actualizar o añadir

Código

producto = {
    "nombre": "Monitor",
    "precio": 200
}

producto["precio"] = 180
producto["stock"] = 5
print(producto)

Qué entrenas

Identifica qué línea actualiza una clave existente y cuál añade una clave nueva.

Reto 5 · ¿Qué patrón de recorrido conviene?

Situación

Quieres mostrar una ficha completa con el nombre del campo y su valor.

Pregunta

¿Te conviene recorrer claves, valores o items()? Explica por qué.

La elección del recorrido también forma parte del razonamiento.

Reto 6 · get() con valor por defecto

Código

config = {
    "tema": "oscuro"
}

print(config.get("idioma", "es"))

Pregunta

¿Qué imprime? ¿Por qué aquí no falla aunque la clave no exista?

Reto 7 · pop() y efecto real

Código

datos = {
    "nombre": "Marta",
    "edad": 28
}

edad = datos.pop("edad")
print(edad)
print(datos)

Pregunta

¿Qué devuelve pop()? ¿Qué queda dentro del diccionario después?

Reto 8 · update() y estructura final

Código

usuario = {
    "nombre": "Luis",
    "activo": True
}

usuario.update({
    "activo": False,
    "email": "luis@email.com"
})

print(usuario)

Prueba obligatoria

Explica qué clave se actualiza, qué clave se añade nueva y cómo queda el diccionario final.

Método rápido para comprobar un diccionario

Qué hacer antes de dar un ejercicio por bien resuelto.

Paso a paso

  1. identifica qué entidad representa
  2. comprueba si las claves tienen sentido
  3. revisa si estás accediendo a una clave que realmente existe
  4. piensa si necesitas leer, cambiar, añadir o recorrer
  5. confirma si una lista podría haber sido mejor o no

Regla útil

Si puedes explicar qué significa cada clave y por qué accedes a ella de esa forma, el diccionario está mucho mejor pensado.
