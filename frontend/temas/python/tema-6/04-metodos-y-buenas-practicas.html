6.4 · Métodos y buenas prácticas: trabajar mejor con diccionarios

Métodos fundamentales

Hay varios métodos muy útiles al trabajar con diccionarios. En este tema vamos a centrarnos en los más prácticos para una base sólida.

keys()

usuario = {
    "nombre": "Ana",
    "edad": 30,
    "activo": True
}

print(usuario.keys())

Devuelve una vista de las claves del diccionario.

values()

print(usuario.values())

Devuelve una vista de los valores.

items()

print(usuario.items())

Devuelve parejas clave-valor.

get() con valor por defecto

config = {
    "tema": "oscuro"
}

print(config.get("idioma", "es"))

Si la clave no existe, devuelve el valor por defecto indicado.

pop(): eliminar y devolver

datos = {
    "nombre": "Marta",
    "edad": 28
}

edad = datos.pop("edad")
print(edad)
print(datos)

Elimina la clave y además devuelve el valor eliminado.

update(): actualizar varios valores

producto = {
    "nombre": "Monitor",
    "precio": 200
}

producto.update({
    "precio": 180,
    "stock": 5
})

print(producto)

Muy útil para modificar varias claves de una vez.

Buenas prácticas

  • usa claves claras y coherentes
  • mantén un mismo criterio de nombres
  • no mezcles datos sin relación en el mismo diccionario
  • usa get() si no estás seguro de que la clave exista
  • piensa si realmente necesitas lista o diccionario

Lista o diccionario: criterio correcto

  • usa lista cuando importa el orden o la secuencia
  • usa diccionario cuando importa el significado de cada dato
Elegir bien la estructura de datos ya es una habilidad de programador serio.

💭 Piensa antes de ejecutar

usuario = {
    "nombre": "Luis",
    "edad": 35
}

usuario.update({"edad": 36, "activo": True})
print(usuario.get("activo"))
print(usuario)
  • ¿qué cambia con update()?
  • ¿qué devuelve get("activo")?

Errores frecuentes

  • usar una lista donde un diccionario sería más claro
  • acceder a claves sin verificar si existen
  • usar nombres de claves inconsistentes
  • pensar que keys() o values() devuelven listas normales siempre

Mini-reto guiado

Crea un diccionario de configuración, actualiza varias claves y elimina una de ellas. 

config = {
    "tema": "claro",
    "idioma": "es",
    "notificaciones": True
}

config.update({
    "tema": "oscuro",
    "fuente": "grande"
})

config.pop("notificaciones")
print(config)

Pregunta de entrevista

¿Cuándo usarías un diccionario en vez de una lista?

Respuesta sólida: cuando los datos tienen nombres o claves con significado y quiero acceder a ellos de forma explícita en lugar de depender de posiciones.
