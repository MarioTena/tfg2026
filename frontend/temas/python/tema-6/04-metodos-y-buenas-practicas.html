Qué vas a entender hoy

Los diccionarios tienen métodos muy útiles para consultar, actualizar y organizar datos de forma más clara y práctica.

Objetivo Métodos útiles

Qué problema resuelve

Trabajar con diccionarios de forma básica está bien, pero sus métodos permiten hacerlo mejor: consultar datos, actualizarlos, eliminarlos o revisar la estructura con menos código y más claridad.

Consultar

Ver claves, valores o parejas completas.

Actualizar

Cambiar varias claves de una sola vez.

Eliminar

Quitar claves que ya no necesitas.

Criterio

Entender cuándo conviene usar diccionario y cuándo lista.

Idea clave: dominar una estructura de datos también significa saber usar bien sus métodos.

Qué debes vigilar

  • no todos los métodos hacen lo mismo
  • algunos consultan y otros modifican
  • get(), pop() y update() tienen funciones distintas
  • también debes saber cuándo usar lista y cuándo diccionario
Piensa así:
elegir bien el método correcto también es parte de programar bien.

Métodos fundamentales

Hay varios métodos muy útiles al trabajar con diccionarios. En este bloque vas a centrarte en los más prácticos para una base sólida.

keys()

usuario = {
    "nombre": "Ana",
    "edad": 30,
    "activo": True
}

print(usuario.keys())

Devuelve una vista de las claves del diccionario.

values()

print(usuario.values())

Devuelve una vista de los valores.

items()

print(usuario.items())

Devuelve parejas clave-valor.

get() con valor por defecto

config = {
    "tema": "oscuro"
}

print(config.get("idioma", "es"))

Si la clave no existe, devuelve el valor por defecto indicado.

pop(): eliminar y devolver

datos = {
    "nombre": "Marta",
    "edad": 28
}

edad = datos.pop("edad")
print(edad)
print(datos)

Elimina la clave y además devuelve el valor eliminado.

update(): actualizar varios valores

producto = {
    "nombre": "Monitor",
    "precio": 200
}

producto.update({
    "precio": 180,
    "stock": 5
})

print(producto)

Muy útil para modificar varias claves de una vez.

Qué hace cada método en la práctica

keys()

Te ayuda a ver qué datos existen.

values()

Sirve cuando te interesan los datos pero no sus nombres.

get()

Es útil cuando una clave puede faltar.

update()

Va bien para varios cambios seguidos sin repetir muchas líneas.

Lista o diccionario: criterio correcto

Usa lista

Cuando importa el orden o la secuencia.

Usa diccionario

Cuando importa el significado de cada dato.

Piensa en acceso

Si vas a pedir datos por nombre, suele encajar mejor un diccionario.

Piensa en estructura

Elegir bien la estructura también es programar bien.

Elegir bien la estructura de datos ya es una habilidad importante de programación.

Cómo pensar qué método necesitas

Paso 1

Pregúntate si solo quieres consultar o también modificar.

Paso 2

Decide si necesitas claves, valores o pares completos.

Paso 3

Si una clave puede faltar, prioriza get().

Paso 4

Si quieres cambiar varias cosas a la vez, piensa en update().

Piensa antes de ejecutar

Antes de lanzar el código, piensa qué método consulta y cuál modifica.

Código

usuario = {
    "nombre": "Luis",
    "edad": 35
}

usuario.update({"edad": 36, "activo": True})
print(usuario.get("activo"))
print(usuario)

Preguntas

  • ¿qué cambia con update()?
  • ¿qué devuelve get("activo")?
Pista: un método modifica y el otro consulta.

Mini-reto guiado

Ahora toca combinar varios métodos sobre un mismo diccionario.

Reto Muy fácil

Tu misión

Crea un diccionario de configuración, actualiza varias claves y elimina una de ellas.

Resolver en playground

Pista

Usa update() para varios cambios y pop() si quieres eliminar una clave concreta.

Mostrar solución 
config = {
    "tema": "claro",
    "idioma": "es",
    "notificaciones": True
}

config.update({
    "tema": "oscuro",
    "fuente": "grande"
})

config.pop("notificaciones")
print(config)

Errores frecuentes

Error 1

Usar una lista donde un diccionario sería más claro.

Error 2

Acceder a claves sin verificar si existen.

Error 3

Usar nombres de claves inconsistentes.

Error 4

Pensar que keys() o values() son siempre listas normales.

Buenas prácticas

Claves coherentes

Usa un criterio uniforme al nombrar datos.

Estructura correcta

No metas en un diccionario cosas que se entenderían mejor como lista.

Método adecuado

Elige el método según la tarea, no por costumbre.

Seguridad

Usa get() cuando no estés seguro de una clave.
