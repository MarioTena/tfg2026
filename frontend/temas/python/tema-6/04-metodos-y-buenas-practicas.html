Qué vas a entender hoy

Los diccionarios tienen métodos muy útiles para consultar, actualizar y organizar datos de forma más clara y práctica.

Objetivo Métodos útiles

Qué problema resuelve

Trabajar con diccionarios de forma básica está bien, pero sus métodos permiten hacerlo mejor: consultar datos, actualizarlos, eliminarlos o revisar la estructura con menos código y más claridad.

Consultar

Ver claves, valores o parejas completas.

Actualizar

Cambiar varias claves de una sola vez.

Eliminar

Quitar claves que ya no necesitas.

Criterio

Entender cuándo conviene usar diccionario y cuándo lista.

Qué debes vigilar

  • no todos los métodos hacen lo mismo

  • también debes saber cuándo usar lista y cuándo diccionario
Pensar bien el método correcto también forma parte de programar bien.

Métodos fundamentales

Hay varios métodos muy útiles al trabajar con diccionarios. En este bloque vas a centrarte en los más prácticos para una base sólida.

keys()

usuario = {
    "nombre": "Ana",
    "edad": 30,
    "activo": True
}

print(usuario.keys())

Devuelve una vista de las claves del diccionario.

values()

print(usuario.values())

Devuelve una vista de los valores.

items()

print(usuario.items())

Devuelve parejas clave-valor.

get() con valor por defecto

config = {
    "tema": "oscuro"
}

print(config.get("idioma", "es"))

Si la clave no existe, devuelve el valor por defecto indicado.

pop(): eliminar y devolver

datos = {
    "nombre": "Marta",
    "edad": 28
}

edad = datos.pop("edad")
print(edad)
print(datos)

Elimina la clave y además devuelve el valor eliminado.

update(): actualizar varios valores

producto = {
    "nombre": "Monitor",
    "precio": 200
}

producto.update({
    "precio": 180,
    "stock": 5
})

print(producto)

Muy útil para modificar varias claves de una vez.

Consultar no es modificar

Consultan

keys(), values(), items(), get()

Modifican

pop(), update(), asignación por clave

Idea útil

Primero decide si solo quieres mirar o también cambiar el diccionario.

Importancia

Evita tocar la estructura por accidente.

Qué hace cada método en la práctica

keys()

Te ayuda a ver qué datos existen.

values()

Sirve cuando te interesan los datos pero no sus nombres.

get()

Es útil cuando una clave puede faltar.

update()

Va bien para varios cambios seguidos sin repetir muchas líneas.

Casos importantes de pop() y update()

pop()

No solo elimina: también devuelve el valor que quita.

update() clave existente

Si la clave ya estaba, su valor se actualiza.

update() clave nueva

Si la clave no estaba, se añade.

Idea práctica

Un mismo método puede servir para dos efectos distintos según el estado del diccionario.

Lista o diccionario: criterio correcto

Usa lista

Cuando importa el orden o la secuencia.

Usa diccionario

Cuando importa el significado de cada dato.

Piensa en acceso

Si vas a pedir datos por nombre, suele encajar mejor un diccionario.

Piensa en estructura

Elegir bien la estructura también es programar bien.

Elegir bien la estructura de datos ya es una habilidad importante de programación.

Errores frecuentes

Error 1

Usar acceso directo cuando la clave puede no existir.

Error 2

Olvidar que pop() modifica el diccionario.

Error 3

No entender si update() está actualizando o añadiendo.

Error 4

Elegir diccionario cuando en realidad solo necesitabas una secuencia simple.

Cómo pensar qué método necesitas

Paso 1

Pregúntate si solo quieres consultar o también modificar.

Paso 2

Decide si necesitas claves, valores o ambas cosas.

Paso 3

Si la clave puede faltar, considera get() o una comprobación con in.

Paso 4

Si necesitas cambiar varias cosas, piensa si update() encaja mejor.
