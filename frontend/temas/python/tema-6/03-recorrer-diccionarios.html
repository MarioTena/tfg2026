Qué vas a entender hoy

Recorrer un diccionario significa pasar por sus claves, sus valores o ambas cosas a la vez según lo que necesites hacer.

Objetivo Recorridos

Qué problema resuelve

Cuando un diccionario tiene varios datos, muchas veces no quieres mirar una sola clave, sino revisarlos todos: mostrarlos, validarlos o procesarlos dentro de un bucle.

Claves

Recorrer los nombres de los datos guardados.

Valores

Clave y valor

Uso real

Mostrar fichas, revisar configuraciones o validar datos.

Idea clave: un diccionario puede recorrerse por claves, por valores o por parejas clave-valor.

Qué debes vigilar

  • recorrer directamente un diccionario da sus claves
  • values() devuelve valores
  • items() devuelve parejas clave-valor
  • debes elegir el patrón según tu objetivo
La pregunta importante no es solo “cómo recorro”, sino “qué necesito recorrer exactamente”.

Qué significa recorrer un diccionario

Igual que recorrías listas para procesar varios elementos, puedes recorrer un diccionario para trabajar con sus claves, sus valores o ambos.

Recorrer claves

persona = {
    "nombre": "Ana",
    "edad": 30,
    "ciudad": "Madrid"
}

for clave in persona:
    print(clave)

Si recorres directamente el diccionario, obtienes sus claves.

Recorrer claves y acceder al valor

persona = {
    "nombre": "Ana",
    "edad": 30,
    "ciudad": "Madrid"
}

for clave in persona:
    print(clave, persona[clave])

Esto permite mostrar cada clave junto con su valor.

Recorrer solo valores con values()

producto = {
    "nombre": "Teclado",
    "precio": 40,
    "stock": 8
}

for valor in producto.values():
    print(valor)

Este patrón es útil cuando el nombre de la clave no importa para la tarea.

Recorrer claves y valores con items()

producto = {
    "nombre": "Teclado",
    "precio": 40,
    "stock": 8
}

for clave, valor in producto.items():
    print(clave, "→", valor)

Esta suele ser la forma más cómoda cuando necesitas ambos.

items() evita tener que escribir diccionario[clave] dentro del bucle.

Cómo decidir qué recorrido usar

Solo claves

Cuando quieres saber qué campos existen.

Solo valores

Cuando los nombres no importan.

Clave y valor

Cuando quieres mostrar o usar ambas cosas a la vez.

Idea práctica

Primero decide qué necesitas y luego elige el método.

Recorrer no siempre es imprimir

Mostrar ficha

Imprimir todos los datos de una entidad.

Validar

Comprobar si algún campo tiene un valor problemático.

Buscar

Revisar si una clave o valor concreto está presente.

Resumir

Construir una vista más clara del contenido.

Casos reales

Ficha de usuario

Mostrar todos sus datos de forma clara.

Configuración

Revisar opciones activadas o guardadas.

Validación

Comprobar si hay datos obligatorios y qué valor tienen.

Informes

Imprimir resúmenes simples de clave y valor.

Errores frecuentes al recorrer

Error 1

No tener claro si necesitas claves, valores o ambos.

Error 2

Recorrer claves y luego olvidar acceder al valor.

Error 3

Usar values() cuando en realidad necesitabas mostrar la clave también.

Error 4

Recorrer sin haber decidido antes para qué sirve el recorrido.

Cómo pensar un recorrido antes de escribirlo

Paso 1

Decide si necesitas claves, valores o ambas cosas.

Paso 2

Elige for clave in diccionario, values() o items().

Paso 3

Piensa qué vas a hacer dentro del bucle.

Paso 4

Usa nombres claros como clave, valor o ambos.
