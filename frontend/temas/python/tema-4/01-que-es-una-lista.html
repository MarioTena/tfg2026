Qué vas a entender hoy

Una lista permite guardar varios valores relacionados dentro de una sola variable, manteniendo el orden.

Objetivo Base para bucles

Qué problema resuelve

Hasta ahora has usado variables sueltas: una para el nombre, otra para la edad, otra para la nota. Pero en programación real muchas veces necesitas trabajar con varios datos del mismo grupo.

Ejemplo real

Los nombres de varios alumnos.

Ejemplo real

Los productos de un pedido.

Ejemplo real

Las temperaturas de varios días.

Ejemplo real

Los mensajes de un chat.

Idea clave: una lista agrupa varios valores relacionados dentro de una sola estructura.

Qué debes recordar

  • se escribe entre []
  • los elementos van separados por comas
  • mantiene el orden
  • puede guardar varios elementos
Piensa así:
una lista no es solo “muchos datos juntos”, sino “muchos datos juntos y ordenados”.

Qué es una lista

Definición

Una lista es una estructura de datos que agrupa varios valores en una sola variable.

Orden

Los elementos mantienen una posición concreta dentro de la lista.

Flexibilidad

Puede contener uno, varios o ningún elemento.

Uso real

Se usa cuando trabajas con grupos de datos relacionados.

Dónde aparecen las listas en problemas reales

Alumnos

Guardar varios nombres, notas o asistencias.

Compras

Representar productos de un carrito o pedido.

Mensajes

Gestionar varias entradas de un chat o notificaciones.

Datos repetidos

Trabajar con varios valores del mismo tipo sin crear muchas variables sueltas.

Cada vez que piensas “necesito guardar varios datos parecidos juntos”, probablemente necesitas una lista.

Ejemplo rápido

Lista de números

numeros = [10, 20, 30, 40]

Lista de nombres

nombres = ["Ana", "Luis", "Marta"]

Lista vacía

tareas = []

Lista con mezcla

mixta = ["Ana", 25, True]

Conceptos clave del tema

El orden importa

No es lo mismo [10, 20, 30] que [30, 20, 10]. En una lista, la posición cambia el significado.

Longitud

Puedes saber cuántos elementos tiene una lista usando len().

Uso real

Las listas se usan muchísimo porque permiten recorrer, añadir, modificar y eliminar elementos.

Buena práctica

Usa nombres claros como nombres, precios o notas.

Cómo pensar una lista antes de crearla

Paso 1

Pregúntate si realmente estás trabajando con varios valores relacionados.

Paso 2

Decide si el orden de esos valores importa.

Paso 3

Piensa si esos valores deberían estar juntos en vez de separados en muchas variables.

Paso 4

Elige un nombre de lista que describa bien el grupo completo.

Muchos errores al principio vienen de no detectar cuándo necesitas una lista y cuándo solo necesitas una variable.

Piensa antes de ejecutar

Antes de probar código, acostúmbrate a predecir qué va a pasar.

Código

colores = ["rojo", "verde", "azul"]
print(colores)
print(len(colores))

Preguntas

  • ¿Qué se imprime en la primera línea?
  • ¿Qué devuelve len(colores)?
Intenta responder antes de ir al playground.

Reto Muy fácil

Tu misión

Crea una lista con 5 ciudades y muestra:

  • la lista completa
  • su longitud con len()
Resolver en playground

Pista

Empieza creando una variable llamada ciudades y guarda dentro varios nombres entre corchetes.

Mostrar solución 
ciudades = ["Madrid", "Sevilla", "Bilbao", "Valencia", "Málaga"]
print(ciudades)
print(len(ciudades))

Errores frecuentes

Error 1

Pensar que una lista es una sola cadena de texto.

Error 2

Olvidar las comas entre elementos.

Error 3

Confundir corchetes [] con paréntesis ().

Error 4

No entender que el orden de los elementos importa.

Buenas prácticas

Nombres claros

Usa nombres que representen el grupo completo: nombres, productos, notas.

Valores relacionados

No metas en una misma lista cosas que no tengan relación entre sí salvo que sea deliberado.

Empieza simple

Primero crea listas pequeñas y entiende bien cómo se representan.

Piensa en el orden

Si el orden importa, la lista suele ser una buena opción.
