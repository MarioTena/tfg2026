Qué debes recordar

Una lista no es solo “muchos datos juntos”, sino “muchos datos juntos y ordenados”.

Qué es una lista

Definición

Una lista es una estructura de datos que agrupa varios valores en una sola variable.

Orden

Los elementos mantienen una posición concreta dentro de la lista.

Flexibilidad

Puede contener uno, varios o ningún elemento.

Uso real

Se usa cuando trabajas con grupos de datos relacionados.

Lista frente a variables sueltas

nombre1 = "Ana"
nombre2 = "Luis"
nombre3 = "Marta"

Esto funciona, pero escala mal. Si quieres recorrer esos nombres, contarlos o modificarlos de forma ordenada, se vuelve incómodo. 

nombres = ["Ana", "Luis", "Marta"]

Con una lista, todos esos valores quedan agrupados en una misma estructura.

Lista frente a string

String

"Ana" representa un texto.

Lista

["Ana", "Luis", "Marta"] representa varios elementos separados.

Diferencia

Un string es una secuencia de caracteres; una lista es una colección de elementos.

Idea útil

No confundas “un texto” con “una colección de textos”.

Dónde aparecen las listas en problemas reales

Alumnos

Guardar varios nombres, notas o asistencias.

Compras

Representar productos de un carrito o pedido.

Mensajes

Gestionar varias entradas de un chat o notificaciones.

Datos repetidos

Trabajar con varios valores del mismo tipo sin crear muchas variables sueltas.

Cada vez que piensas “necesito guardar varios datos parecidos juntos”, probablemente necesitas una lista.

Ejemplo rápido

Lista de números

numeros = [10, 20, 30, 40]

Lista de nombres

nombres = ["Ana", "Luis", "Marta"]

Lista vacía

tareas = []

Lista con mezcla

mixta = ["Ana", 25, True]

Casos especiales que debes entender

Lista vacía

No tiene elementos, pero sigue siendo una lista válida.

Un solo elemento

También sigue siendo una lista: [5].

Repetidos

Una lista puede contener el mismo valor varias veces.

Orden

[1, 2, 3] no es lo mismo que [3, 2, 1].

Conceptos clave del tema

El orden importa

No es lo mismo [10, 20, 30] que [30, 20, 10]. En una lista, la posición cambia el significado.

Longitud

Puedes saber cuántos elementos tiene una lista usando len().

Uso real

Las listas se usan muchísimo porque permiten recorrer, añadir, modificar y eliminar elementos.

Buena práctica

Usa nombres claros como nombres, precios o notas.

Cómo pensar una lista antes de crearla

Paso 1

Pregúntate si realmente estás trabajando con varios valores relacionados.

Paso 2

Decide si el orden de esos valores importa.

Paso 3

Piensa si esos valores deberían estar juntos en vez de separados en muchas variables.

Paso 4

Elige un nombre de lista que describa bien el grupo completo.

Muchos errores al principio vienen de no detectar cuándo necesitas una lista y cuándo solo necesitas una variable.

Piensa antes de ejecutar

Antes de probar código, acostúmbrate a predecir qué va a pasar.
