Qué vas a dominar hoy

Recorrer una lista significa pasar por sus elementos uno a uno para leerlos, mostrarlos, procesarlos, contarlos o transformarlos.

for Listas Patrón básico

Por qué es importante

Esta es una habilidad central en programación. Muchas tareas reales consisten en repetir una misma operación sobre varios datos.

Idea clave: cuando tienes varios elementos, casi siempre aparece la necesidad de recorrerlos.

Qué debes ser capaz de hacer

  • mostrar todos los elementos
  • sumar valores
  • contar cuántos cumplen una condición
  • buscar un valor concreto
  • decidir si necesitas elemento o índice

Dónde aparece recorrer listas en problemas reales

Facturas

Sumar importes o aplicar impuestos a varios productos.

Alumnos

Contar aprobados, mostrar nombres o calcular medias.

Búsquedas

Comprobar si un valor aparece dentro de una colección.

Transformaciones

Aplicar la misma operación a todos los elementos de una lista.

Cuando una tarea afecta a muchos datos parecidos, recorrer una lista suele ser la solución natural.

Recorrer con for

frutas = ["manzana", "pera", "uva"]

for fruta in frutas:
    print(fruta)

En cada iteración, la variable fruta toma el siguiente elemento de la lista.

Ventaja de este enfoque: no necesitas pensar en índices para los casos más simples. Python te da directamente cada elemento.

Formas comunes de recorrer una lista

Recorrer no siempre significa hacer lo mismo. El patrón cambia según tu objetivo.

Mostrar

Leer cada valor y sacarlo por pantalla o usarlo en otro paso.

Acumular

Ir sumando o combinando valores en una variable acumuladora.

Contar

Revisar una condición y aumentar un contador cuando se cumple.

Buscar

Comprobar si un valor concreto aparece dentro de la lista.

Recorrer listas para resolver tareas reales

Cuando recorres una lista, normalmente no lo haces “porque sí”. Lo haces para conseguir algo: sumar, contar, buscar o transformar datos.

Acumular Contar Buscar

1. Recorrer para acumular

Un patrón muy común es usar una lista y un acumulador. En cada iteración, vas sumando o combinando el valor actual con el resultado anterior. 

numeros = [10, 20, 30, 40]
suma = 0

for numero in numeros:
    suma = suma + numero

print(suma)

Aquí se recorre la lista para sumar todos sus elementos.

La variable suma empieza en 0 porque al principio todavía no has procesado ningún elemento.

2. Recorrer para contar condiciones

A veces no quieres sumar valores, sino contar cuántos cumplen una condición concreta. En ese caso usas un contador. 

numeros = [2, 5, 8, 11, 14]
contador_pares = 0

for numero in numeros:
    if numero % 2 == 0:
        contador_pares = contador_pares + 1

print(contador_pares)

Aquí no se suma el valor, se cuenta cuántos son pares.

Un acumulador guarda cantidades o resultados; un contador solo aumenta cuando se cumple algo.

3. Recorrer para buscar un valor

Otra tarea muy habitual es comprobar si un valor aparece en la lista. En este caso vas revisando elemento a elemento hasta encontrarlo. 

nombres = ["Ana", "Luis", "Marta", "Carlos"]

for nombre in nombres:
    if nombre == "Marta":
        print("Encontrado")

También puedes combinar esto con break si quieres parar en cuanto lo encuentres.

Qué tienen en común estos tres casos

  • en todos recorres la lista elemento a elemento
  • lo que cambia no es el bucle, sino lo que haces dentro
  • la pregunta importante es: ¿quieres sumar, contar o buscar?
Cuando entiendes este patrón, empiezas a ver que muchos ejercicios cambian de enunciado, pero comparten la misma estructura mental.

Elemento o índice: elegir bien

Esta decisión aparece muchas veces y conviene entenderla bien.

Cuando solo necesitas el valor

Usa for elemento in lista. Es la opción más simple y legible.

No uses índices por costumbre si no los necesitas. Complica la lectura sin aportar valor.

Cuando necesitas la posición

A veces sí necesitas saber en qué posición está cada elemento. En esos casos puedes usar range(len(lista)). 

frutas = ["manzana", "pera", "uva"]

for i in range(len(frutas)):
    print(i, frutas[i])

Esto muestra el índice y el valor.

Cómo pensar un recorrido antes de escribirlo

Paso 1

Pregúntate qué quieres hacer con cada elemento: mostrar, sumar, contar o buscar.

Paso 2

Decide si necesitas el valor directamente o también su posición.

Paso 3

Elige si necesitas un acumulador, un contador o solo recorrer.

Paso 4

Revisa cuántas iteraciones habrá: una por cada elemento de la lista.

Antes de escribir un bucle, identifica el patrón. Eso reduce muchísimo los errores.

Piensa antes de ejecutar

Antes de lanzar el código, intenta seguir mentalmente el recorrido del bucle.

Código

numeros = [3, 6, 9]

for numero in numeros:
    print(numero * 2)

Preguntas

  • ¿qué se imprime en cada iteración?
  • ¿cuántas iteraciones hay?
Pista: hay tantas iteraciones como elementos tenga la lista.

Error típico: confundir el elemento con el índice

frutas = ["manzana", "pera", "uva"]

for fruta in frutas:
    print(frutas[fruta])

Esto está mal, porque fruta contiene valores como "manzana", no índices numéricos.

Si recorres directamente una lista con for fruta in frutas, la variable contiene el valor, no la posición.

Mini-reto guiado

Ahora toca aplicar el patrón de recorrido a una situación práctica.

Reto Muy fácil

Tu misión

Recorre una lista de precios y muestra cada precio con IVA calculado al 21%.

Resolver en playground

Pista

Aquí no necesitas posiciones, solo el valor de cada precio.

Mostrar solución 
precios = [10, 20, 30]

for precio in precios:
    print(precio * 1.21)

Errores frecuentes

Error 1

Confundir el elemento con el índice.

Error 2

No inicializar un acumulador o contador antes del bucle.

Error 3

Usar índices cuando no hacen falta.

Error 4

No entender qué hace el bloque dentro de cada iteración.

Buenas prácticas

Nombres claros

Si recorres notas, usa nota, no x.

Separa roles

Diferencia claramente acumuladores, contadores y variables del bucle.

Elige la forma simple

Si no necesitas índices, recorre directamente los elementos.

Lee el patrón

Antes de escribir, piensa si vas a mostrar, sumar, contar o buscar.
