Qué vas a dominar hoy

Recorrer una lista significa pasar por sus elementos uno a uno para leerlos, mostrarlos, procesarlos, contarlos o transformarlos.

for Listas Patrón básico

Qué problema resuelve

Muchas tareas reales no consisten en trabajar con un solo dato, sino en repetir una acción sobre varios: mostrar, sumar, buscar, contar o transformar elementos de una colección.

Mostrar

Ver todos los elementos de una lista uno a uno.

Sumar

Acumular valores a medida que avanzas por la lista.

Contar

Comprobar cuántos cumplen una condición.

Buscar

Revisar si un valor aparece dentro de la colección.

Idea clave: cuando tienes varios elementos, casi siempre aparece la necesidad de recorrerlos.

Qué debes recordar

  • un for recorre la lista elemento a elemento
  • puedes usarlo para mostrar, sumar, contar o buscar
  • si no necesitas posición, no necesitas índices
  • el patrón del bucle es parecido; cambia lo que haces dentro
Recorrer una lista no es un objetivo en sí: es el medio para resolver una tarea concreta.

Cuando una tarea afecta a muchos datos parecidos, recorrer una lista suele ser la solución natural.

Recorrer con for

frutas = ["manzana", "pera", "uva"]

for fruta in frutas:
    print(fruta)

En cada iteración, la variable fruta toma el siguiente elemento de la lista.

Ventaja de este enfoque: no necesitas pensar en índices para los casos más simples. Python te da directamente cada elemento.

Formas comunes de recorrer una lista

Recorrer no siempre significa hacer lo mismo. El patrón cambia según tu objetivo.

Mostrar

Leer cada valor y sacarlo por pantalla o usarlo en otro paso.

Acumular

Ir sumando o combinando valores en una variable acumuladora.

Contar

Revisar una condición y aumentar un contador cuando se cumple.

Buscar

Comprobar si un valor concreto aparece dentro de la lista.

Recorrer listas para resolver tareas reales

Cuando recorres una lista, normalmente no lo haces “porque sí”. Lo haces para conseguir algo: sumar, contar, buscar o transformar datos.

Acumular Contar Buscar

1. Recorrer para acumular

Un patrón muy común es usar una lista y un acumulador. En cada iteración, vas sumando o combinando el valor actual con el resultado anterior. 

numeros = [10, 20, 30, 40]
suma = 0

for numero in numeros:
    suma = suma + numero

print(suma)

Aquí se recorre la lista para sumar todos sus elementos.

La variable suma empieza en 0 porque al principio todavía no has procesado ningún elemento.

2. Recorrer para contar condiciones

A veces no quieres sumar valores, sino contar cuántos cumplen una condición concreta. En ese caso usas un contador. 

numeros = [2, 5, 8, 11, 14]
contador_pares = 0

for numero in numeros:
    if numero % 2 == 0:
        contador_pares = contador_pares + 1

print(contador_pares)

Aquí no se suma el valor, se cuenta cuántos son pares.

Un acumulador guarda cantidades o resultados; un contador solo aumenta cuando se cumple algo.

3. Recorrer para buscar un valor

Otra tarea muy habitual es comprobar si un valor aparece en la lista. En este caso vas revisando elemento a elemento hasta encontrarlo. 

nombres = ["Ana", "Luis", "Marta", "Carlos"]

for nombre in nombres:
    if nombre == "Marta":
        print("Encontrado")

También puedes combinar esto con break si quieres parar en cuanto lo encuentres.

Qué tienen en común estos tres casos

  • en todos recorres la lista elemento a elemento
  • lo que cambia no es el bucle, sino lo que haces dentro
  • la lista es la estructura; el patrón lo marca tu objetivo

¿Recorrer elementos o índices?

Elementos directos

Es la forma más simple cuando solo necesitas el valor.

Índices

Son útiles cuando también necesitas saber la posición que ocupa cada elemento.

Regla práctica

Si no necesitas posición, recorre directamente los elementos.

Precaución

No compliques un recorrido si el problema no te pide trabajar con índices.
