Qué entrenas aquí

En este bloque no solo usas listas. Aprendes a leerlas, prever cómo cambian, detectar errores y justificar si una operación trabaja por valor o por posición.

Lectura Depuración Práctica guiada

Objetivo

Estos retos te ayudan a entender mejor qué representa una lista y qué ocurre realmente cuando accedes, modificas, recorres o aplicas métodos.

Idea clave: una operación sobre listas no consiste solo en “que funcione”, sino en saber qué parte de la estructura cambia y por qué.

Cómo usar esta pantalla

  • lee el código antes de ejecutarlo
  • predice qué se imprime o cómo queda la lista
  • distingue entre posición, valor y tramo
  • detecta si hay error de lógica o de índice
Si no puedes explicar qué representa la lista y qué cambia tras una línea de código, todavía no controlas del todo la estructura.

Reto 1 · ¿Qué imprime?

Código

frutas = ["manzana", "pera", "uva"]
print(frutas[0])
print(frutas[-1])

Qué debes pensar

¿Qué valor representa el índice 0? ¿Y cuál representa el índice -1?

Reto 2 · Detecta el error de índice

Código incorrecto

numeros = [10, 20, 30]
print(numeros[3])

Pregunta

¿Por qué falla ese acceso? ¿Qué índices son realmente válidos en esa lista?

La longitud de una lista no coincide con su último índice.

Reto 3 · Modificar una posición

Código

colores = ["rojo", "verde", "azul"]
colores[1] = "amarillo"
print(colores)

Qué debes pensar

¿Se ha añadido un elemento nuevo o se ha sustituido uno que ya existía?

Reto 4 · Slicing

Código

numeros = [10, 20, 30, 40, 50]
print(numeros[1:4])

Pregunta

¿Qué elementos devuelve? ¿Por qué aparece 40 pero no aparece 50?

Reto 5 · Recorrer y acumular

Código

numeros = [5, 10, 15]
suma = 0

for numero in numeros:
    suma = suma + numero

print(suma)

Qué entrenas

Seguir cómo cambia un acumulador mientras se recorren los elementos de la lista.

Reto 6 · remove() o pop()

Código A

frutas = ["manzana", "pera", "uva"]
frutas.remove("pera")

Código B

frutas = ["manzana", "pera", "uva"]
frutas.pop(1)
Los dos eliminan el mismo elemento en este caso, pero no piensan igual: uno trabaja por valor y el otro por posición.

Reto 7 · Ordenar una lista

Código

numeros = [8, 3, 1, 9, 4]
numeros.sort()
print(numeros)

Pregunta

¿Qué imprime? ¿La lista original se mantiene igual o cambia después de usar sort()?

Reto 8 · Buscar un valor

Código

nombres = ["Ana", "Luis", "Marta"]

for nombre in nombres:
    if nombre == "Marta":
        print("Encontrado")

Prueba obligatoria

Piensa qué ocurre si el valor está una vez, varias veces o no está.

Método rápido para comprobar una operación sobre listas

Qué hacer antes de dar un ejercicio por terminado.

Paso a paso

  1. identifica qué representa la lista
  2. distingue si trabajas con posición o valor
  3. pregúntate si el código lee, modifica o corta una parte
  4. simula el resultado final
  5. comprueba si hay riesgo de índice incorrecto

Regla útil

Cuando trabajas con listas, casi siempre estás haciendo una de estas cosas: crear, acceder, modificar, recorrer, buscar, contar, añadir, eliminar u ordenar.
