A veces no esperas simplemente una excepción de Python: quieres indicar tú mismo que un dato no es válido y que el programa no debería seguir por ese camino.

Qué problema resuelve

Hay entradas que quizá no provocan una excepción automática de Python, pero sí rompen las reglas de tu programa: edades negativas, notas fuera de rango, precios imposibles o divisores no permitidos.

Idea clave: raise te permite señalar explícitamente que algo no cumple las reglas del programa.

Qué debes vigilar

  • raise sirve para lanzar una excepción

  • el mensaje debe explicar bien el problema

Piensa así: con raise no reaccionas a un fallo externo, sino que decides frenar tú mismo un caso inválido.

Notas fuera de rango, edades negativas o precios imposibles.

Campos obligatorios

Validar antes de calcular o continuar con datos incorrectos.

raise y las validaciones aparecen cuando quieres señalar de forma explícita que un dato incumple las reglas del programa.

¿Qué es raise?

raise sirve para lanzar una excepción de forma intencional cuando detectas que un dato no cumple una regla. 

edad = -5

if edad < 0:
    raise ValueError("La edad no puede ser negativa")

Por qué esto es útil

  • evita que el programa siga con datos incorrectos
  • hace tus validaciones más claras
  • mejora la calidad del código
  • te obliga a pensar qué entradas son válidas

Capturar una excepción vs lanzarla

Capturar

Reaccionas a una excepción que ya ha aparecido, usando except.

Lanzar

Provocas tú mismo una excepción con raise al detectar un dato inválido.

Capturar

Piensas “si esto falla, ¿qué hago?”.

Lanzar

Piensas “si esto incumple mi regla, no debería continuar”.

Validar antes de continuar

nota = 15

if nota < 0 or nota > 10:
    raise ValueError("La nota debe estar entre 0 y 10")

Más casos de validación útiles

Texto vacío

if nombre == "":
    raise ValueError("El nombre no puede estar vacío")

Divisor inválido

if b == 0:
    raise ValueError("El divisor no puede ser cero")

Valor no permitido

if opcion not in ["A", "B", "C"]:
    raise ValueError("Opción no permitida")

Rango

if edad < 0 or edad > 120:
    raise ValueError("Edad fuera de rango")

Mensajes de error buenos y malos

Mensaje vago

"Error"

Mensaje útil

"La nota debe estar entre 0 y 10"

Regla útil

El mensaje debería explicar qué regla se ha incumplido.

Objetivo

Que quien lea el error entienda el problema sin adivinar.

Combinar raise con funciones

def dividir(a, b):
    if b == 0:
        raise ValueError("El divisor no puede ser cero")
    return a / b

Este patrón es muy útil: la función valida antes de calcular.

Funciones más seguras gracias a la validación

Cuando validas dentro de una función, dejas más claro qué espera esa función y qué casos no acepta. Eso hace el código más robusto y más fácil de reutilizar.

Capturar un error provocado con raise

try:
    edad = -1
    if edad < 0:
        raise ValueError("Edad inválida")
except ValueError as error:
    print(error)

Prevenir, capturar o lanzar

Prevenir

Validas una condición con if antes de que aparezca un problema.

Capturar

Usas except cuando una excepción puede aparecer al ejecutar.

Lanzar

Usas raise cuando tú decides que un dato incumple una regla.

Idea clave

No son lo mismo y conviene saber cuándo usar cada enfoque.

Cómo pensar una validación con raise

Paso 1

Define qué datos son válidos.

Paso 2

Escribe la condición que detecta lo inválido.

Paso 3

Lanza una excepción clara si esa regla se rompe.

Paso 4

Solo continúa si los datos cumplen la regla.

Piensa antes de ejecutar

Antes de lanzar el programa, decide si el dato cumple la regla o debería provocar un error.

Código

def validar_precio(precio):
    if precio < 0:
        raise ValueError("Precio inválido")
    return precio

print(validar_precio(10))

Preguntas

  • ¿qué pasa con 10?
  • ¿qué pasaría con -3?
  • ¿por qué aquí tiene sentido frenar el proceso?

Errores frecuentes

  • usar raise sin pensar qué regla se está validando
  • no elegir un mensaje claro
  • seguir el programa con datos incorrectos cuando deberías frenarlo
  • confundir lanzar una excepción con capturarla

Buenas prácticas

Valida con intención

No lances errores por costumbre, sino porque una regla real se incumple.

Usa mensajes útiles

Explican claramente qué condición no se ha cumplido.

Hazlo pronto

Conviene validar antes de seguir calculando con datos inválidos.

Piénsalo en funciones

Hace que las funciones sean más seguras y explícitas.
