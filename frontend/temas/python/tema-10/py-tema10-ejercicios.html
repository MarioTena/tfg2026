Cómo trabajar esta pantalla

Aquí no basta con que el programa funcione cuando todo va bien. Debes pensar qué ocurre cuando algo falla y cómo responder a ese error.

try / except raise Validación

Recomendación

  • piensa primero qué podría fallar
  • distingue entre prevenir, capturar y lanzar errores
  • fíjate en qué bloque debe ejecutar cada caso
  • abre el playground y prueba también entradas incorrectas

Clave del tema

Un programa robusto no es el que nunca falla, sino el que sabe reaccionar bien cuando algo falla.
