try y except

Qué vas a entender hoy

Con try y except puedes anticipar errores probables y definir cómo debe reaccionar el programa.

Objetivo Reacción controlada

Qué problema resuelve

Hay situaciones donde algo puede salir mal aunque tu programa esté bien escrito: conversiones inválidas, archivos ausentes o datos mal introducidos.

Conversión

Un texto puede no ser convertible a número.

Archivos

El archivo que buscas puede no existir.

Entradas

El usuario puede escribir algo incorrecto.

Continuidad

El programa puede responder mejor en lugar de romperse.

Idea clave: try y except no evitan todos los errores, pero te permiten controlarlos mejor.

Qué debes vigilar

  • try contiene el código que podría fallar
  • except define qué hacer si falla
  • capturar errores concretos suele ser mejor que usar un except genérico
  • no deberías usar except para esconder problemas sin entenderlos
Piensa así:
gestionar errores no es taparlos, sino responder a ellos con intención.

Idea principal

Con try y except puedes intentar ejecutar código que podría fallar, y definir qué hacer si efectivamente aparece un error. 

try:
    numero = int("hola")
except:
    print("Ha ocurrido un error")
try significa “inténtalo”; except significa “si falla, actúa así”.

Por qué es útil

Evita cortes bruscos

El programa no tiene por qué terminar de golpe.

Mejora mensajes

Puedes mostrar una explicación más clara.

Valida entradas

Encaja muy bien con datos del usuario.

Uso real

Es muy común con conversiones y archivos.

Ejemplo con conversión de tipos

texto = "123"

try:
    numero = int(texto)
    print(numero)
except:
    print("No se pudo convertir")

Capturar errores concretos

Aunque puedes usar un except genérico, normalmente es mejor capturar el tipo concreto de error. 

try:
    numero = int("hola")
except ValueError:
    print("El texto no representa un número")

Esto hace el código más claro y más profesional.

Varios except

try:
    archivo = open("datos.txt", "r")
    numero = int("hola")
except FileNotFoundError:
    print("El archivo no existe")
except ValueError:
    print("El valor no se puede convertir")

Cada bloque maneja un tipo distinto de problema.

Guardar el error en una variable

try:
    numero = int("hola")
except ValueError as error:
    print("Ha ocurrido un error:", error)

Esto es útil para depuración y para entender mejor qué ha pasado.

Error típico: usar except para ocultarlo todo

Un except demasiado genérico puede esconder errores que en realidad deberías corregir. 

try:
    algo = 10 / 0
except:
    print("Error")

Esto funciona, pero a veces impide entender qué tipo de error ocurrió realmente.

Cómo pensar un bloque try / except

Paso 1

Detecta qué parte del código podría fallar de verdad.

Paso 2

Piensa qué tipo de error esperas.

Paso 3

Decide qué mensaje o reacción tiene sentido.

Paso 4

Captura lo más concreto posible.

Piensa antes de ejecutar

Intenta seguir el flujo del programa antes de verlo en pantalla.

Código

texto = "abc"

try:
    numero = int(texto)
    print("Conversión correcta")
except ValueError:
    print("Conversión incorrecta")

Pista: el fallo ocurre al intentar convertir un texto no numérico.

Mini-reto guiado

Ahora toca gestionar una entrada inválida sin romper el programa.

Reto Muy fácil

Tu misión

Pide al usuario un número y usa try / except para mostrar un mensaje si introduce texto inválido.

Solución de referencia

texto = input("Escribe un número: ")

try:
    numero = int(texto)
    print("Número válido:", numero)
except ValueError:
    print("Debes escribir un número entero")

Errores frecuentes

Error 1

Usar except genérico por costumbre.

Error 2

No pensar qué error concreto puede producirse.

Error 3

Capturar errores para ocultarlos sin entenderlos.

Error 4

Meter demasiado código dentro del try.

Buenas prácticas

Captura errores concretos

Tu código será más claro y preciso.

Usa mensajes útiles

Explican mejor qué ha pasado.

No tapes todo

Una excepción no debería ocultar un fallo de diseño.

Anticipa

Piensa qué entradas o situaciones pueden fallar.
