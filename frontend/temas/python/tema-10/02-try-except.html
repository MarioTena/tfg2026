Qué vas a entender hoy

Con try y except puedes anticipar errores probables y definir cómo debe reaccionar el programa.

Objetivo Reacción controlada

Qué problema resuelve

Hay situaciones donde algo puede salir mal aunque tu programa esté bien escrito: conversiones inválidas, archivos ausentes o datos mal introducidos.

Conversión

Un texto puede no ser convertible a número.

Archivos

El archivo que buscas puede no existir.

Entradas

El usuario puede escribir algo incorrecto.

Continuidad

El programa puede responder mejor en lugar de romperse.

Idea clave: try y except no evitan todos los errores, pero te permiten controlarlos mejor.

Qué debes vigilar

  • try contiene el código que podría fallar
  • except define qué hacer si falla
  • capturar errores concretos suele ser mejor que usar un except genérico
  • no deberías usar except para esconder problemas sin entenderlos
Piensa así: gestionar errores no es taparlos, sino responder a ellos con intención.

Dónde aparecen try y except en problemas reales

Conversión de datos

Cuando un texto puede no convertirse correctamente a número.

Archivos

Cuando un archivo puede no existir o no abrirse como esperas.

Entradas del usuario

Cuando el usuario puede escribir algo incorrecto o inesperado.

Continuidad

Cuando quieres responder al fallo sin romper el programa de golpe.

try y except aparecen cuando sabes que una operación puede fallar y quieres controlar mejor la reacción del programa.

Idea principal

Con try y except puedes intentar ejecutar código que podría fallar, y definir qué hacer si efectivamente aparece una excepción. 

try:
    numero = int("hola")
except:
    print("Ha ocurrido un error")
try significa “inténtalo”; except significa “si falla, actúa así”.

Por qué es útil

Evita cortes bruscos

El programa no tiene por qué terminar de golpe.

Mejora mensajes

Puedes mostrar una explicación más clara.

Valida entradas

Encaja muy bien con datos del usuario.

Uso real

Es muy común con conversiones y archivos.

Ejemplo con conversión de tipos

texto = "123"

try:
    numero = int(texto)
    print(numero)
except:
    print("No se pudo convertir")

Capturar errores concretos

Aunque puedes usar un except genérico, normalmente es mejor capturar el tipo concreto de error. 

try:
    numero = int("hola")
except ValueError:
    print("El texto no representa un número")

Esto hace el código más claro y más profesional.

Varios except

try:
    archivo = open("datos.txt", "r")
    numero = int("hola")
except FileNotFoundError:
    print("El archivo no existe")
except ValueError:
    print("El valor no se puede convertir")

Cada bloque maneja un tipo distinto de problema.

Guardar el error en una variable

try:
    numero = int("hola")
except ValueError as error:
    print("Ha ocurrido un error:", error)

Esto es útil para depuración y para entender mejor qué ha pasado.

Qué parte debe ir dentro del try

Dentro del try conviene poner solo el código que realmente podría lanzar la excepción que esperas.

Meter demasiado código dentro del try puede hacer más difícil entender qué parte ha fallado.

Ejemplo: try demasiado grande

try:
    texto = input("Escribe un número: ")
    numero = int(texto)
    print(numero)
    print(10 / numero)
except ValueError:
    print("No era un número válido")

Aquí se mezclan posibles fallos distintos. El código funciona, pero no es la forma más clara de razonar.

except genérico frente a específico

Genérico

Captura cualquier excepción, incluso algunas que no esperabas.

Específico

Deja claro qué problema estás anticipando.

Genérico

Puede ocultar fallos de diseño.

Específico

Suele ser más claro, más útil y más fácil de mantener.

Capturar no sustituye a pensar la lógica

El manejo de excepciones ayuda a responder a fallos previsibles, pero no sustituye a diseñar bien el programa ni a validar las reglas importantes cuando corresponda.

Piensa antes de ejecutar

Intenta seguir el flujo del programa antes de verlo en pantalla.

Código

texto = "abc"

try:
    numero = int(texto)
    print("Conversión correcta")
except ValueError:
    print("Conversión incorrecta")
