else, finally y errores comunes

Qué vas a entender hoy

Además de try y except, Python permite usar else y finally para controlar mejor qué pasa cuando algo falla y cuando no falla.

Objetivo Flujo completo

Qué problema resuelve

A veces no basta con capturar un error. También quieres ejecutar algo solo si todo fue bien, o algo que deba ocurrir siempre, haya o no haya fallo.

else

Se ejecuta solo si no hubo excepción.

finally

Se ejecuta siempre, pase lo que pase.

Limpieza

Muy útil para cierres o pasos finales.

Claridad

Hace más explícito el flujo del programa.

Idea clave: except responde al error, else responde al éxito y finally responde al cierre del proceso.

Qué debes vigilar

  • else solo entra si no hubo excepción
  • finally se ejecuta siempre
  • no todos los errores son del mismo tipo
  • conocer excepciones comunes te ayuda a leer mejor los fallos
Piensa así:
un bloque de excepciones bien escrito no solo controla errores: también deja claro el recorrido normal y el recorrido excepcional.

Bloque else en excepciones

El bloque else se ejecuta solo si no ha ocurrido ninguna excepción. 

try:
    numero = int("10")
except ValueError:
    print("No válido")
else:
    print("Conversión correcta")

Bloque finally

El bloque finally se ejecuta siempre, haya o no error. 

try:
    archivo = open("datos.txt", "r")
except FileNotFoundError:
    print("No existe el archivo")
finally:
    print("Fin del intento")
finally es útil cuando hay acciones de cierre o limpieza que deben ocurrir pase lo que pase.

Estructura completa

try:
    numero = int("5")
except ValueError:
    print("Error de conversión")
else:
    print("Todo bien")
finally:
    print("Bloque final")

Errores comunes que ya conoces

ValueError

Valor con formato incorrecto.

TypeError

Tipos incompatibles en una operación.

NameError

Variable no definida.

IndexError

Índice fuera de rango.

KeyError

Clave inexistente.

FileNotFoundError

Archivo ausente.

ZeroDivisionError

División entre cero.

Ejemplo con división

try:
    resultado = 10 / 0
except ZeroDivisionError:
    print("No se puede dividir entre cero")

Cómo leer el flujo completo

try

Contiene el intento principal.

except

Se activa si ocurre un error capturado.

else

Solo corre si no hubo error.

finally

Siempre corre al final.

Piensa antes de ejecutar

Sigue mentalmente el recorrido del programa y decide qué bloques se activan.

Código

try:
    numero = int("7")
except ValueError:
    print("Error")
else:
    print("Correcto")
finally:
    print("Siempre aparece")

Preguntas

  • ¿qué bloques se ejecutan?
  • ¿por qué no entra en except?
  • ¿por qué finally aparece siempre?
Pista: la conversión aquí sí es válida.

Mini-reto guiado

Ahora toca usar éxito, error y cierre en una misma estructura.

Reto Muy fácil

Tu misión

Haz un programa que intente abrir un archivo. Si existe, muestre un mensaje en else. Al final, siempre debe mostrar un mensaje final con finally.

Ejemplo de referencia

try:
    archivo = open("datos.txt", "r")
except FileNotFoundError:
    print("El archivo no existe")
else:
    print("Archivo abierto correctamente")
finally:
    print("Fin del proceso")

Errores frecuentes

Error 1

Pensar que else se ejecuta aunque haya error.

Error 2

No entender que finally aparece siempre.

Error 3

Confundir tipos de excepción comunes.

Error 4

No pensar el flujo completo del bloque.

Buenas prácticas

Separa casos

Distingue claramente error, éxito y cierre.

Conoce excepciones comunes

Te ayuda a diagnosticar mejor.

No improvises

Piensa qué debe pasar en cada escenario.

Hazlo legible

Un flujo claro se entiende y se mantiene mejor.
