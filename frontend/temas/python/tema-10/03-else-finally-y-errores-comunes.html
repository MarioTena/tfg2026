else, finally y errores comunes

Qué vas a entender hoy

Además de try y except, Python permite usar else y finally para controlar mejor qué pasa cuando algo falla y cuando no falla.

Objetivo Flujo completo

Qué problema resuelve

A veces no basta con capturar un error. También quieres ejecutar algo solo si todo fue bien, o algo que deba ocurrir siempre, haya o no haya fallo.

else

Se ejecuta solo si no hubo excepción.

finally

Se ejecuta siempre, pase lo que pase.

Limpieza

Muy útil para cierres o pasos finales.

Claridad

Hace más explícito el flujo del programa.

Idea clave: except responde al error, else responde al éxito y finally responde al cierre del proceso.

Qué debes vigilar

  • else solo entra si no hubo excepción
  • finally se ejecuta siempre
  • no todos los errores son del mismo tipo
  • conocer excepciones comunes te ayuda a leer mejor los fallos
Piensa así: un bloque de excepciones bien escrito no solo controla errores, también deja claro el recorrido normal y el recorrido excepcional.

Dónde aparecen else y finally en problemas reales

Éxito claro

Usar else cuando una acción solo debe ocurrir si no hubo error.

Cierre garantizado

Usar finally cuando algo debe ejecutarse pase lo que pase.

Limpieza

Dejar el proceso en un estado coherente al terminar.

Flujo legible

Separar claramente error, éxito y cierre para entender mejor el programa.

else y finally aparecen cuando no solo quieres controlar el error, sino también dejar claro qué pasa si todo sale bien y qué debe ejecutarse siempre al final.

Bloque else en excepciones

El bloque else se ejecuta solo si no ha ocurrido ninguna excepción. 

try:
    numero = int("10")
except ValueError:
    print("No válido")
else:
    print("Conversión correcta")

Por qué else aporta claridad

else te permite separar claramente el camino donde todo salió bien del camino donde hubo error. Eso hace el código más legible.

Regla útil: si algo debe ocurrir solo cuando no hubo excepción, else suele ser un buen sitio para ponerlo.

Bloque finally

El bloque finally se ejecuta siempre, haya o no error. 

try:
    archivo = open("datos.txt", "r")
except FileNotFoundError:
    print("No existe el archivo")
finally:
    print("Fin del intento")
finally es útil cuando hay acciones de cierre o limpieza que deben ocurrir pase lo que pase.

Cuándo tiene sentido real usar finally

Cierre

Acciones que deben ocurrir siempre al final.

Limpieza

Dejar el proceso en un estado coherente.

Mensajes finales

Indicar que el intento terminó, con o sin error.

Idea útil

No es un adorno: tiene sentido cuando algo debe ejecutarse siempre.

Estructura completa

try:
    numero = int("5")
except ValueError:
    print("Error de conversión")
else:
    print("Todo bien")
finally:
    print("Bloque final")

Cómo leer el flujo completo

try

Contiene el intento principal.

except

Se activa si ocurre un error capturado.

else

Solo corre si no hubo error.

finally

Siempre corre al final.

Errores comunes que ya conoces

ValueError

Valor con formato incorrecto.

TypeError

Tipos incompatibles en una operación.

NameError

Variable no definida.

IndexError

Índice fuera de rango.

KeyError

Clave inexistente.

FileNotFoundError

Archivo ausente.

ZeroDivisionError

División entre cero.

Importancia

Conocer nombres comunes te ayuda a interpretar fallos reales con más rapidez.

Ejemplo con división

try:
    resultado = 10 / 0
except ZeroDivisionError:
    print("No se puede dividir entre cero")

Qué debería ir en cada bloque

En try

La operación que podría fallar.

En except

La respuesta al error capturado.

En else

Lo que solo tiene sentido si todo fue bien.

En finally

Lo que debe ocurrir siempre al final.

Piensa antes de ejecutar

Sigue mentalmente el recorrido del programa y decide qué bloques se activan.

Código

try:
    numero = int("7")
except ValueError:
    print("Error")
else:
    print("Correcto")
finally:
    print("Siempre aparece")
