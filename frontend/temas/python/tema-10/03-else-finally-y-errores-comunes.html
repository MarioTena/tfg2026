10.3 · else, finally y errores comunes: controlar mejor el flujo

Bloque else en excepciones

El bloque else se ejecuta solo si no ha ocurrido ninguna excepción. 

try:
    numero = int("10")
except ValueError:
    print("No válido")
else:
    print("Conversión correcta")

Bloque finally

El bloque finally se ejecuta siempre, haya o no error. 

try:
    archivo = open("datos.txt", "r")
except FileNotFoundError:
    print("No existe el archivo")
finally:
    print("Fin del intento")
finally es útil cuando hay acciones de cierre o limpieza que deben ocurrir pase lo que pase.

Estructura completa

try:
    numero = int("5")
except ValueError:
    print("Error de conversión")
else:
    print("Todo bien")
finally:
    print("Bloque final")

Errores comunes que ya conoces

  • ValueError: valor con formato incorrecto
  • TypeError: operación con tipos incompatibles
  • NameError: variable no definida
  • IndexError: índice fuera de rango
  • KeyError: clave inexistente
  • FileNotFoundError: archivo ausente
  • ZeroDivisionError: división entre cero

Ejemplo con división

try:
    resultado = 10 / 0
except ZeroDivisionError:
    print("No se puede dividir entre cero")

💭 Piensa antes de ejecutar

try:
    numero = int("7")
except ValueError:
    print("Error")
else:
    print("Correcto")
finally:
    print("Siempre aparece")
