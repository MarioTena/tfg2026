Qué vas a entender hoy

Un error no es solo “algo que sale mal”: es una señal que te dice en qué parte del programa hay un problema y de qué tipo es.

Objetivo Entender fallos

Qué problema resuelve

Hasta ahora has trabajado sobre todo en situaciones donde el programa funcionaba bien. Pero en la práctica real también hay conversiones inválidas, archivos que no existen, índices incorrectos o reglas mal planteadas.

Entradas inválidas

Un usuario puede escribir algo inesperado.

Archivos ausentes

El programa puede intentar abrir algo que no existe.

Índices erróneos

Puedes intentar acceder a una posición que no existe.

Lógica incorrecta

El programa puede funcionar, pero devolver un resultado mal planteado.

Idea clave: no todos los errores son iguales. Entender su tipo cambia completamente la forma de corregirlos.

Qué debes vigilar

  • no todos los fallos son errores de sintaxis
  • algunos aparecen al ejecutar, no al escribir
  • otros no rompen el programa, pero producen resultados incorrectos
  • leer bien el tipo de error ayuda mucho más que asustarse
Piensa así:
un mensaje de error no es un ataque: es información útil que te ayuda a localizar el problema.

Por qué este tema importa de verdad

Hasta ahora has aprendido a escribir programas que funcionan en condiciones normales. Pero en la práctica, los programas reales también deben saber reaccionar cuando algo sale mal.

Un programador principiante intenta que nunca haya errores. Un programador sólido asume que algunos errores ocurrirán y diseña cómo responder a ellos.

Qué es un error

Un error es una situación en la que el programa no puede continuar normalmente o no produce el comportamiento correcto que se esperaba.

Puede deberse a código incorrecto, datos no válidos, archivos ausentes o conversiones imposibles.

Tipos generales de problemas

Error de sintaxis

El programa ni siquiera puede interpretarse.

Error en ejecución

Aparece mientras el programa ya está corriendo.

Error lógico

El programa corre, pero hace algo incorrecto.

Lectura correcta

No todos se corrigen del mismo modo.

Error de sintaxis

if 5 > 3
    print("Hola")

Aquí falta :. Python no puede ni empezar a ejecutar el programa.

Error en tiempo de ejecución

numero = int("hola")

El programa empieza, pero falla al intentar convertir un texto no numérico a entero.

Error lógico

precio = 100
descuento = 10
final = precio - descuento / 100
print(final)

El programa funciona, pero el cálculo no representa el descuento que seguramente se pretendía.

Algunos errores muy comunes en Python

ValueError

El valor tiene un formato inválido.

TypeError

Estás mezclando tipos incompatibles.

NameError

La variable no existe.

IndexError

El índice está fuera del rango válido.

KeyError

La clave no existe en el diccionario.

FileNotFoundError

El archivo que intentas abrir no existe.

Qué significa gestionar errores

Gestionar errores significa detectar situaciones problemáticas y responder de forma controlada, en lugar de dejar que el programa se rompa sin más.

La gestión de errores no elimina todos los problemas, pero evita que el programa reaccione de forma brusca o confusa.

Cómo pensar un error antes de corregirlo

Paso 1

Mira si el programa ni siquiera arranca.

Paso 2

Si arranca, revisa en qué línea falla.

Paso 3

Lee el tipo de error antes de tocar el código.

Paso 4

Piensa si el fallo es técnico o lógico.

Piensa antes de ejecutar

Antes de lanzar el código, intenta clasificar el problema en tu cabeza.

Código

edad = int("veinte")

Preguntas

  • ¿funcionará?
  • ¿qué tipo de error provocará?
  • ¿por qué no puede hacerse esa conversión?
Pista: el problema no está en la sintaxis, sino en el valor usado.

Mini-reto guiado

Ahora toca distinguir correctamente varios tipos de fallo.

Reto Muy fácil

Tu misión

Clasifica estos casos como error de sintaxis, tiempo de ejecución o lógico:

  • usar una variable que no existe
  • olvidar un paréntesis
  • calcular mal una fórmula aunque el programa corra

Solución razonada

  • usar una variable que no existe → error en tiempo de ejecución
  • olvidar un paréntesis → error de sintaxis
  • calcular mal una fórmula aunque el programa corra → error lógico
Clasificar bien el fallo te ahorra mucho tiempo al corregirlo.
