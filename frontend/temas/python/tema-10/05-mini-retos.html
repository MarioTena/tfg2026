Qué entrenas aquí

Aquí vas a practicar cómo se comporta un programa cuando algo falla y cómo decidir si conviene prevenir, capturar o lanzar una excepción.

try / except raise Criterio

Objetivo

Vas a dejar de ver las excepciones como simples bloques de sintaxis y empezar a pensar qué puede fallar, qué regla se incumple y qué comportamiento quieres ofrecer.

Idea clave: un programa robusto no ignora los errores; los anticipa, los interpreta y decide cómo reaccionar.

Cómo usar esta pantalla

  • lee la situación o el código
  • piensa qué puede fallar exactamente
  • decide qué excepción tiene sentido capturar
  • pregúntate si deberías validar antes o lanzar una excepción tú mismo
No uses excepciones solo porque sí. Piensa qué error real quieres controlar y qué comportamiento quieres ofrecer.

Reto 8 · Prevenir, capturar o lanzar

Situación

Tienes una entrada del usuario, una conversión y una regla de negocio.

Prueba obligatoria

Identifica qué parte conviene prevenir, cuál capturar y cuál lanzar.

Cómo pensar este tipo de ejercicios

Antes de escribir un try, identifica el fallo real que puede ocurrir.

Paso a paso

  1. piensa qué puede fallar exactamente
  2. decide si ese fallo es una excepción de Python o una regla tuya
  3. si es una excepción previsible, captúrala con el tipo adecuado
  4. si es una regla de negocio inválida, valídala y usa raise si hace falta
  5. usa else y finally cuando aporten claridad

Regla útil

Controlar errores no consiste en taparlos, sino en reaccionar de forma útil y comprensible.
