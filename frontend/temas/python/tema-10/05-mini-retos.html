Qué entrenas aquí

Aquí vas a practicar cómo se comporta un programa cuando algo falla y cómo responder de forma segura.

try / except raise Validación

Objetivo

Vas a empezar a controlar errores frecuentes y a validar datos para que el programa no dependa de que todo salga perfecto.

Idea clave: un programa robusto no ignora los errores; los anticipa y los gestiona.

Cómo usar esta pantalla

  • lee el reto
  • piensa qué puede fallar
  • decide qué excepción tiene sentido controlar
  • pruébalo en el playground y fuerza errores
No uses excepciones solo porque sí. Piensa qué error real quieres controlar y qué comportamiento quieres ofrecer al usuario.

Misiones rápidas

Mini-reto 1 · Conversión segura

Controla el caso en el que la entrada no sea numérica.

texto = input("Escribe un número: ")

try:
    numero = int(texto)
    print("Número correcto")
except ValueError:
    print("Entrada no válida")

Mini-reto 2 · División segura

Evita que el programa se rompa al dividir entre cero.

try:
    a = 10
    b = 0
    print(a / b)
except ZeroDivisionError:
    print("No se puede dividir entre cero")

Mini-reto 3 · Abrir archivo

Controla el caso en el que el archivo no exista.

try:
    with open("datos.txt", "r") as archivo:
        print(archivo.read())
except FileNotFoundError:
    print("Archivo no encontrado")

Mini-reto 4 · Validación con raise

Lanza una excepción cuando el dato no cumple la regla.

def validar_nota(nota):
    if nota < 0 or nota > 10:
        raise ValueError("Nota inválida")
    return nota

Mini-reto 5 · else y finally

Distingue entre éxito, error y cierre del proceso.

try:
    numero = int("7")
except ValueError:
    print("Error")
else:
    print("Todo correcto")
finally:
    print("Fin del proceso")

Mini-reto 6 · Capturar varias excepciones

Distingue entre distintos tipos de fallo.

try:
    texto = "abc"
    numero = int(texto)
    print(10 / numero)
except ValueError:
    print("No era un número válido")
except ZeroDivisionError:
    print("No se puede dividir entre cero")

Cómo pensar este tipo de ejercicios

Antes de escribir un try, identifica el fallo real que puede ocurrir.

Paso a paso

  1. piensa qué puede fallar exactamente
  2. controla la excepción adecuada
  3. no mezcles errores distintos sin motivo
  4. usa raise cuando quieras imponer una regla
  5. usa else y finally cuando aporten claridad

Regla útil

Controlar errores no consiste en taparlos, sino en reaccionar de forma útil y comprensible.

Pregunta típica de entrevista

Pregunta: ¿Para qué sirve raise si ya existe except?

Respuesta sólida: raise permite lanzar una excepción cuando detectas una regla inválida; except sirve para capturar y gestionar una excepción cuando ocurre.
