7.3 · Métodos de strings: transformar y analizar texto

Por qué importan los métodos de strings

Los strings aparecen en casi todos los programas: nombres, formularios, búsquedas, mensajes, validaciones. Por eso Python ofrece muchos métodos para trabajar con ellos.

lower() y upper()

texto = "PyThOn"
print(texto.lower())
print(texto.upper())

Muy útiles para comparar textos sin depender de mayúsculas y minúsculas.

strip()

texto = "   hola   "
print(texto.strip())

Elimina espacios al principio y al final. Muy útil con entradas de usuario.

replace()

texto = "Hola mundo"
print(texto.replace("mundo", "Python"))

Sustituye una parte del texto por otra.

find()

texto = "programacion"
print(texto.find("gra"))

Devuelve la posición donde empieza la subcadena, o -1 si no la encuentra.

startswith() y endswith()

archivo = "datos.csv"
print(archivo.startswith("da"))
print(archivo.endswith(".csv"))

Muy útiles para validaciones y comprobaciones simples.

split()

frase = "uno dos tres"
print(frase.split())

Convierte el string en una lista, separándolo normalmente por espacios.

join()

palabras = ["Hola", "mundo"]
print(" ".join(palabras))

Une elementos de una lista en un único string.

⚠️ Importante: los strings son inmutables

Métodos como lower() o replace() no cambian el string original; devuelven uno nuevo. 

texto = "HOLA"
texto.lower()
print(texto)

Esto seguirá imprimiendo HOLA. Si quieres conservar el cambio, debes guardarlo. 

texto = texto.lower()

💭 Piensa antes de ejecutar

texto = "  PyThOn  "
texto = texto.strip().lower()
print(texto)
  • ¿qué hace cada método?
  • ¿por qué se pueden encadenar?

Errores frecuentes

  • pensar que los métodos modifican el string original
  • olvidar guardar el resultado
  • usar find() sin comprobar el caso -1
  • no distinguir entre split() y join()

Mini-reto guiado

Crea una frase con espacios sobrantes y conviértela a minúsculas sin espacios laterales. 

frase = "   HOLA PYTHON   "
frase = frase.strip().lower()
print(frase)

Pregunta de entrevista

¿Por qué conviene recordar que los strings son inmutables al usar métodos como replace() o lower()?

Respuesta sólida: porque esos métodos no cambian el string original; devuelven uno nuevo, así que si no lo guardas, pierdes el resultado.
