Qué vas a entender hoy

Si un string es una secuencia, entonces puedes acceder a posiciones concretas y extraer partes del texto igual que hacías con otras estructuras.

Objetivo Acceso y fragmentos

Qué problema resuelve

Muchas veces no necesitas todo el texto, sino una letra concreta, el último carácter o un fragmento. La indexación y el slicing sirven para eso.

Idea clave: acceder a una posición y extraer un fragmento no son exactamente la misma operación.

Qué debes vigilar

  • los índices empiezan en 0
  • los índices negativos cuentan desde el final
  • el slicing no incluye el límite final
  • un índice fuera de rango provoca error
Piensa así: acceder es pedir una posición exacta; hacer slicing es pedir un tramo del texto.

Dónde aparecen la indexación y el slicing en problemas reales

Iniciales

Tomar la primera letra o un carácter concreto de un texto.

Códigos

Extraer una parte concreta de una cadena con formato fijo.

Fragmentos

Recuperar solo una parte de un mensaje o dato más largo.

Revisión

Acceder al principio, al final o a una zona concreta del texto.

La indexación y el slicing aparecen cuando necesitas precisión: no todo el texto, sino una posición o un fragmento concreto.

Acceder a una posición concreta

Igual que en las listas, los strings usan índices que empiezan en 0. 

texto = "Python"
print(texto[0])
print(texto[1])

Esto imprimirá P y y.

Un índice devuelve un string de un carácter

texto = "Hola"
letra = texto[0]
print(letra)
print(type(letra))

El resultado no es “algo especial”: sigue siendo un string, pero de longitud 1.

Índices negativos

También puedes acceder desde el final. 

texto = "Python"
print(texto[-1])
print(texto[-2])
  • -1 es el último carácter
  • -2 es el penúltimo

Error típico: índice fuera de rango

texto = "sol"
print(texto[3])

Esto da error porque los índices válidos son 0, 1 y 2.

Slicing: extraer una parte del string

El slicing permite tomar un fragmento de texto. 

texto = "Python"
print(texto[1:4])

Devuelve yth. Empieza en el índice 1 y llega hasta antes del 4.

Índice frente a slicing

Índice

Devuelve un solo carácter.

Slicing

Devuelve un fragmento, aunque tenga una sola letra o incluso esté vacío.

Ejemplo 1

texto[2]

Ejemplo 2

texto[2:3]

Slicing desde el inicio o hasta el final

texto = "Python"
print(texto[:3])
print(texto[3:])

Esto devuelve Pyt y hon.

Copiar un string completo

texto = "Python"
print(texto[:])

Devuelve todo el string.

Salto en slicing

texto = "Python"
print(texto[0:6:2])

Va tomando caracteres de dos en dos.

Invertir un string

texto = "Python"
print(texto[::-1])

Esto devuelve el texto al revés.

Casos frontera útiles

Primer carácter

texto[0]

Último carácter

texto[-1]

Slicing vacío

texto[2:2]

Todo el string

texto[:]

Matiz importante: índice fuera del límite vs slicing fuera del límite

texto = "sol"
# texto[10] da error
print(texto[10:20])

Un índice fuera de rango falla, pero un slicing fuera del límite simplemente devuelve lo que pueda o incluso una cadena vacía.

Este detalle es muy útil para no confundir errores de acceso con cortes seguros.

Cómo pensar indexación y slicing antes de escribirlos

Paso 1

Decide si quieres un carácter o un fragmento.

Paso 2

Si quieres una posición exacta, usa un índice.

Paso 3

Si quieres un tramo, usa slicing.

Paso 4

Recuerda que el límite final del slicing no se incluye.
