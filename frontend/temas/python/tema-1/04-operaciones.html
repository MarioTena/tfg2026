Qué vas a entender hoy

Python no solo muestra texto o guarda datos. También puede calcular. En este bloque vas a aprender cómo funcionan las operaciones básicas y qué debes observar para no confundirte.

Objetivo Base matemática

Qué problema resuelve

En cuanto empiezas a trabajar con números, necesitas hacer operaciones: sumar, restar, multiplicar, dividir o comparar resultados. Python permite hacer esos cálculos de forma directa, pero debes entender bien qué hace cada operador.

Sumar

Combinar cantidades o acumular valores.

Restar

Calcular diferencias entre números.

Multiplicar

Escalar un valor o repetir una cantidad.

Dividir

Repartir o calcular cocientes y restos.

Idea clave: no se trata de memorizar símbolos aislados. Se trata de entender qué resultado produce cada operación.

Qué debes vigilar

  • qué operador estás usando
  • si Python devuelve entero o decimal
  • el orden en el que se resuelven las operaciones
  • si estás mezclando tipos de datos incompatibles
Un cálculo mal entendido no siempre da error. A veces funciona, pero da un resultado distinto del que esperabas.

Python también calcula

Igual que una calculadora, Python puede resolver operaciones matemáticas básicas. 

print(5 + 3)

Python evalúa la operación y muestra el resultado.

Operaciones básicas

a = 10
b = 3

print(a + b)   # suma
print(a - b)   # resta
print(a * b)   # multiplicación
print(a / b)   # división
print(a // b)  # división entera
print(a % b)   # resto
print(a ** b)  # potencia

No necesitas memorizar todo de golpe. Lo importante ahora es entender el efecto general de cada operador.

Piensa antes de ejecutar

print(10 / 3)
  • ¿Esperas un 3?
  • ¿Esperas un decimal?
  • ¿Por qué?
Acostúmbrate a prever el resultado antes de ejecutar. Esa costumbre vale más que ir probando sin pensar.

División normal vs división entera

print(10 / 3)
print(10 // 3)

/

Devuelve un resultado decimal.

//

Devuelve solo la parte entera del cociente.

%

Devuelve el resto de la división.

**

Eleva un número a una potencia.

La diferencia entre división normal, entera y resto aparece constantemente en problemas reales.

Orden de operaciones

print(2 + 3 * 4)

Python respeta las reglas matemáticas habituales:

  • primero multiplicación y división
  • después suma y resta
  • los paréntesis cambian el orden
print((2 + 3) * 4)
Si no piensas el orden de evaluación, puedes obtener resultados correctos técnicamente, pero no los que querías.

Error típico: mezclar texto y número

edad = 20
print("Tengo " + edad + " años")

Este código falla porque Python no puede sumar directamente un string y un entero.

Forma correcta:

print("Tengo", edad, "años")
No todo lo que parece una “suma” lo es. Con texto y números hay que pensar también en los tipos de datos.

Dónde aparece esta idea en problemas reales

Precios

Sumar importes, aplicar descuentos o calcular totales.

Contadores

Incrementar o reducir cantidades paso a paso.

Medias y repartos

Dividir cantidades y analizar resultados.

Reglas

Aplicar fórmulas simples dentro de un programa.
