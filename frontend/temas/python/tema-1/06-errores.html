Qué vas a entender hoy

Programar no consiste en no equivocarse. Consiste en entender qué ha pasado cuando algo falla y usar esa información para corregir el código.

Objetivo Lectura de errores

Qué problema resuelve

Cuando Python encuentra algo incorrecto, no se queda “pensando”. Se detiene y te da una pista. Aprender a leer esa pista es una habilidad básica.

Sintaxis

Errores por escribir algo mal: comillas, paréntesis o signos.

Nombre

Errores por usar variables que no existen.

Tipo

Errores por mezclar datos incompatibles.

Índice

Errores por intentar acceder a posiciones que no existen.

Idea clave: un error no es una señal de que “no sirves”. Es información técnica sobre algo concreto que debes revisar.

Qué debes vigilar

  • el tipo de error que aparece
  • la línea donde se produce
  • el mensaje final de Python
  • si el problema es de sintaxis, de nombre, de tipo o de acceso
El principiante mira el error y se bloquea. El programador aprende a leerlo como una pista.

Antes de empezar: equivocarte es normal

Los errores no significan que no sabes programar. Significan que estás escribiendo código real y que el ordenador te está devolviendo feedback.

Un buen programador no es el que nunca falla, sino el que sabe interpretar el fallo y corregirlo.

Error 1 · Variable no definida

print(nombre)
nombre = "Ana"

Este código falla porque intentas usar una variable antes de crearla. 

NameError: name 'nombre' is not defined
Python ejecuta de arriba hacia abajo. Si en una línea una variable aún no existe, no puede usarla.

Error 2 · Comillas mal cerradas

print("Hola mundo)

Aquí falta cerrar correctamente el texto. 

SyntaxError: unterminated string literal
Si abres comillas, debes cerrarlas. Muchas veces el error es más simple de lo que parece.

Error 3 · Mezclar texto y número

edad = 20
print("Tengo " + edad + " años")

Este error aparece porque intentas unir un string con un entero usando +. 

TypeError: can only concatenate str (not "int") to str

Forma correcta:

print("Tengo", edad, "años")

Error 4 · División entre cero

x = 10
y = 0
print(x / y)
ZeroDivisionError: division by zero

Matemáticamente no se puede dividir entre cero, y Python bloquea esa operación.

Error 5 · Índice fuera de rango

nombre = "Ana"
print(nombre[5])
IndexError: string index out of range

El string tiene menos posiciones de las que intentas acceder.

Si el texto tiene 3 letras, no puedes pedir una posición que no existe.

Cómo leer un error

Traceback (most recent call last):
  File "programa.py", line 2, in <module>
    print(nombre)
NameError: name 'nombre' is not defined

Cuando veas algo así, fíjate en tres cosas:

  • el tipo de error
  • la línea donde ocurre
  • el mensaje final, que suele ser la pista más útil
El mensaje final no está ahí para decorar. Normalmente te dice exactamente qué revisar.
