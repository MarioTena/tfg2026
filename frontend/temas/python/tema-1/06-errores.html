1.6 · Laboratorio de errores: aprender equivocándote

💡 Antes de empezar

Los errores no significan que no sabes programar. Significan que estás programando.

Un buen programador no es el que no comete errores, sino el que sabe leerlos.

🔴 Error 1 · Variable no definida

print(nombre)
nombre = "Ana"

❓ Antes de ejecutar:

  • ¿Qué crees que pasará?
  • ¿Por qué?
Python ejecuta de arriba hacia abajo.

Mensaje esperado:

NameError: name 'nombre' is not defined

El error te está diciendo exactamente el problema: intentas usar algo que aún no existe.

🔴 Error 2 · Comillas mal cerradas

print("Hola mundo)

❓ ¿Qué falta?

SyntaxError: unterminated string literal
Si abres comillas, debes cerrarlas.

🔴 Error 3 · Mezclar texto y número

edad = 20
print("Tengo " + edad + " años")

❓ ¿Por qué falla?

TypeError: can only concatenate str (not "int") to str

Python no puede sumar directamente texto y número.

Usa comas en print o convierte el número a texto.

🔴 Error 4 · División entre cero

x = 10
y = 0
print(x / y)
ZeroDivisionError: division by zero

Matemáticamente no existe dividir entre cero. Python lo bloquea.

🔴 Error 5 · Índice fuera de rango

nombre = "Ana"
print(nombre[5])
IndexError: string index out of range
Intentas acceder a una posición que no existe.

🧠 Cómo leer un error (muy importante)

Traceback (most recent call last):
  File "programa.py", line 2, in <module>
    print(nombre)
NameError: name 'nombre' is not defined

Siempre fíjate en:

  • 🔎 Tipo de error (NameError, TypeError...)
  • 📍 Línea donde ocurre
  • 🧾 Mensaje final (es la clave)
El mensaje final casi siempre te dice exactamente qué está mal.

🎯 Reto final del laboratorio

Crea un programa que tenga:

  • Una variable mal escrita
  • Una división entre cero
  • Un índice incorrecto

Ejecuta uno a uno y observa los errores.

Tu objetivo no es que funcione. Tu objetivo es entender el mensaje.
