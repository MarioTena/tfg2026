Qué vas a entender hoy

Un set es una colección de elementos únicos. No está pensado para posiciones, sino para pertenencia y ausencia de duplicados.

Idea clave: un set está pensado para trabajar con elementos únicos, no con posiciones.

Qué debes vigilar

  • un set no garantiza orden fijo
  • no admite duplicados
  • no se usa con índices como una lista
  • un set vacío se crea con set()
Piensa así:
si lo importante es que algo esté o no esté, y sin repetir, probablemente estés pensando en un set.

Qué es un set

Un set es una colección no ordenada de elementos únicos. Su característica más importante es que no admite duplicados. 

numeros = {1, 2, 3, 4}
letras = {"a", "b", "c"}

Qué ocurre con duplicados

valores = {1, 2, 2, 3, 3, 3}
print(valores)

El resultado solo conserva una copia de cada valor.

Un set es muy útil para eliminar repetidos de forma sencilla.

Diferencia con una lista

Lista

Mantiene orden y puede tener duplicados.

Set

No garantiza orden y elimina duplicados.

Lista

Encaja si quieres recorrer posiciones.

Set

Encaja si quieres pertenencia y unicidad.

Crear un set vacío

conjunto = set()

No debes usar {} para esto, porque eso crea un diccionario vacío.

Añadir y eliminar elementos

add()

colores = {"rojo", "verde"}
colores.add("azul")
print(colores)

Añade un nuevo elemento al set.

remove()

colores = {"rojo", "verde", "azul"}
colores.remove("verde")
print(colores)

Elimina un elemento existente.

Comprobar pertenencia

numeros = {1, 2, 3, 4}
print(3 in numeros)
print(10 in numeros)

Los sets son especialmente buenos para comprobaciones de pertenencia rápidas.

Cuándo pensar en un set

Sin repetidos

Cuando no quieres duplicados.

No importa orden

Cuando el orden no forma parte del problema.

Buscar rápido

Cuando quieres comprobar pertenencia muchas veces.

Cuando luego quieres intersectar o unir conjuntos.
