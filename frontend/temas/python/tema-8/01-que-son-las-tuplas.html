Las tuplas aparecen cuando importa el orden, pero también quieres transmitir estabilidad y evitar cambios accidentales.

Qué es una tupla

Una tupla es una estructura de datos ordenada, parecida a una lista, pero con una diferencia fundamental: no se modifica directamente después de crearla. 

coordenada = (10, 20)
colores = ("rojo", "verde", "azul")

Tupla frente a lista

Ambas

Son secuencias ordenadas.

Ambas

Permiten indexación, longitud y slicing.

Lista

Es mutable y se usa bien cuando el contenido cambia.

Tupla

Es inmutable y expresa mejor datos fijos o estables.

Por qué importa que sea inmutable

La tupla no solo “no cambia”: también comunica una intención de diseño. Si usas una tupla, le estás diciendo a quien lea tu código que ese grupo de datos no debería modificarse.

A veces una lista podría funcionar técnicamente, pero una tupla representa mejor el problema.

Casos simples importantes

Tupla vacía

vacia = ()

Un solo elemento

una = (7,)

Tipos mezclados

datos = ("Ana", 30, True)

Idea útil

Una tupla puede ser corta, larga o incluso estar vacía.

La tupla de un solo elemento

una_tupla = (5,)

La coma es obligatoria. Sin ella, Python no lo interpreta como tupla de un elemento. 

no_es_tupla = (5)

Acceder a sus elementos

colores = ("rojo", "verde", "azul")
print(colores[0])
print(colores[-1])

Igual que en listas y strings, puedes acceder por índice positivo o negativo.

Slicing en tuplas

numeros = (10, 20, 30, 40)
print(numeros[1:3])

Una tupla sigue siendo una secuencia, así que también puedes cortar fragmentos.

Casos reales donde una tupla encaja bien

Coordenadas

(x, y) o (latitud, longitud).

Fechas simples

Un grupo ordenado como (dia, mes, anio).

Pares fijos

Datos que tienen sentido como conjunto estable.

Retornos múltiples

Varias piezas de información devueltas por una función.

Cuándo una lista funcionaría pero una tupla representa mejor

Podrías guardar unas coordenadas en una lista, pero si el valor representa una posición fija, la tupla comunica mejor que ese grupo no está pensado para ir cambiando elemento a elemento.

Error típico: intentar modificar una tupla

colores = ("rojo", "verde", "azul")
colores[1] = "amarillo"

Esto da error porque la tupla es inmutable.

Cómo pensar una tupla antes de usarla

Paso 1

Pregúntate si el orden importa.

Paso 2

Pregúntate si esos datos deberían permanecer estables.

Paso 3

Si el grupo es fijo y ordenado, una tupla suele encajar mejor.

Paso 4

Si necesitas cambiar, añadir o eliminar con frecuencia, probablemente quieras una lista.
