Qué vas a entender hoy

Elegir bien entre lista, tupla o set no es un detalle menor: cambia cómo representas el problema y cómo se entiende tu código.

Objetivo Elegir bien

Qué problema resuelve

No basta con que el programa funcione. También importa que la estructura elegida represente bien lo que estás modelando: datos ordenados, fijos, únicos o modificables.

Lista

Para secuencias ordenadas que pueden cambiar.

Tupla

Para grupos ordenados que deben mantenerse estables.

Set

Para valores únicos donde el orden no importa.

Claridad

Elegir bien mejora la legibilidad y la intención del código.

Idea clave: la estructura correcta no se elige por costumbre, sino por la naturaleza del problema.

Qué debes vigilar

  • si importa el orden
  • si puede haber duplicados
  • si los datos van a cambiar
  • si lo importante es pertenencia o acceso por posición
Piensa así:
antes de escribir código, pregúntate qué representa mejor tus datos en el mundo real.

La decisión correcta importa

Elegir bien la estructura de datos es una habilidad importante. No se trata solo de que el programa funcione, sino de que represente bien el problema.

Usa lista cuando...

  • importa el orden
  • admites duplicados
  • vas a modificar elementos
tareas = ["estudiar", "practicar", "repasar"]

Usa tupla cuando...

  • importa el orden
  • no quieres cambios
  • el conjunto de datos es fijo
coordenada = (40.4, -3.7)

Usa set cuando...

  • no quieres duplicados
  • la pertenencia es importante
  • quieres comparar grupos de elementos
categorias = {"python", "web", "datos"}

Regla general

Paso 1

Paso 2

Paso 3

Paso 4

Piensa antes de ejecutar

Aquí no se trata de ejecutar código, sino de razonar qué estructura encaja mejor en cada caso.

Preguntas

  • ¿en cuáles quieres evitar duplicados?
  • ¿cuál representa mejor datos fijos?
Pista: piensa primero en la naturaleza del dato, no en la sintaxis.

La clave no es memorizar, sino justificar por qué esa estructura encaja mejor.

Errores frecuentes

Error 1

Elegir siempre lista por costumbre.

Error 2

No pensar si los datos cambiarán o no.

Error 3

Usar set cuando realmente necesitas orden fijo.

Error 4

Usar tupla cuando luego quieres modificar los datos.

Buenas prácticas

Representa bien el dato

La estructura debe reflejar la lógica del problema.

No uses una sola por costumbre

Cada estructura resuelve necesidades distintas.

Piensa antes de programar

Una buena elección simplifica el código posterior.

Hazlo legible

La estructura correcta también ayuda a otros a entender tu intención.
