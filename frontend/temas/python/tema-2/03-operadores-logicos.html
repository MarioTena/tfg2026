Qué vas a entender hoy

Muchas decisiones reales no dependen de una sola condición. En este bloque aprendes a combinar reglas.

Condiciones Lógica compuesta

Qué problema resuelve

En la vida real, casi nunca basta una sola pregunta. A menudo necesitas comprobar dos cosas a la vez, aceptar varias posibilidades o negar una situación.

Acceso

Entrar solo si la contraseña es correcta y la cuenta está activa.

Promoción

Bloqueo

Mostrar error si no hay stock o el producto está inactivo.

Control

Negar acceso si no ha iniciado sesión.

Idea clave: en problemas reales, lo habitual no es una sola condición, sino varias condiciones combinadas.

Qué debes vigilar

  • si deben cumplirse todas las reglas o solo una
  • que cada parte del or sea una comparación completa
  • la prioridad entre operadores
  • la legibilidad de la expresión
Piensa así:
una condición larga pero mal planteada vale menos que una condición simple y clara.

Por qué hacen falta operadores lógicos

Una sola condición suele quedarse corta. Los operadores lógicos permiten unir preguntas pequeñas para construir una decisión más completa.

Reglas simultáneas

Necesitas que se cumplan varias cosas a la vez.

Opciones alternativas

Te vale cualquiera de varias condiciones posibles.

Negación

Quieres detectar que algo no ocurre o no se cumple.

Casos reales

Login, permisos, promociones, validaciones y filtros usan esta lógica constantemente.

Los tres operadores lógicos

and

Ambas condiciones deben ser verdaderas.

or

Basta con que una de las condiciones sea verdadera.

not

Invierte el valor lógico.

Resumen práctico

and exige más, or permite más, not niega.

Dónde aparecen and, or y not en problemas reales

Login

Contraseña correcta y cuenta activa.

Promociones

Descuento por edad o por ser estudiante.

Bloqueos

Error por falta de stock o producto inactivo.

Permisos

Acceso permitido si es admin o tiene un permiso especial.

Cuando una decisión depende de varias reglas al mismo tiempo, aparecen los operadores lógicos.

Los tres bloques clave

Entiende qué exige cada operador antes de usarlo.

and

edad = 20
tiene_entrada = True

if edad >= 18 and tiene_entrada:
    print("Puedes pasar")
else:
    print("No puedes pasar")

Para que el resultado sea verdadero, deben cumplirse las dos condiciones.

or

es_admin = False
es_supervisor = True

if es_admin or es_supervisor:
    print("Acceso permitido")
else:
    print("Acceso denegado")

Con or, basta con que una de las condiciones sea verdadera.

not

cuenta_activa = False

if not cuenta_activa:
    print("La cuenta está inactiva")

not invierte el valor lógico: convierte True en False y viceversa.

Es útil cuando quieres expresar que algo no se cumple.

Tabla mental rápida

and

True and True → True

True and False → False

False and True → False

False and False → False

or

True or False → True

False or True → True

False or False → False

not

not True → False

not False → True

Lectura rápida

and pide todo, or acepta una opción, not niega.

Combinar comparaciones

edad = 25

if edad >= 18 and edad <= 30:
    print("Está dentro del rango")

Este tipo de lógica es muy común para validar intervalos, permisos y reglas simultáneas.

Forma corta para rangos

edad = 25

if 18 <= edad <= 30:
    print("Está dentro del rango")

Esta forma es más idiomática en Python y suele ser más fácil de leer.

Dónde se cometen más errores

Las condiciones compuestas fallan sobre todo por planteamiento.

Error muy común con or

color = "rojo"

if color == "rojo" or "azul":
    print("Color permitido")

Está mal planteado porque la segunda parte no es una comparación completa. 

color = "rojo"

if color == "rojo" or color == "azul":
    print("Color permitido")
Cada lado del or debe ser una condición completa.

Prioridad de operadores

Python sigue un orden:

  1. not
  2. and
  3. or

Si la expresión es compleja, usa paréntesis para dejar clara tu intención. 

edad = 20
es_socio = False

if edad >= 18 and (es_socio or edad > 65):
    print("Condición cumplida")

Cómo leer una condición larga sin equivocarte

Paso 1

Divide la expresión en partes pequeñas.

Paso 2

Evalúa cada comparación por separado.

Paso 3

Mira si están unidas por and o or.

Paso 4

Fíjate si hay paréntesis.

Paso 5

Decide el resultado final solo al final del proceso.

edad = 20
es_socio = False
es_vip = True

if edad >= 18 and (es_socio or es_vip):
    print("Acceso especial")

Razonar una condición paso a paso

edad = 17
tiene_permiso = True

if edad >= 16 and tiene_permiso:
    print("Puede participar")

Pregunta 1

¿edad >= 16? Sí.

Pregunta 2

¿tiene_permiso es verdadero? Sí.

Pregunta 3

¿Ambas se cumplen al mismo tiempo? Sí.

Resultado

La condición completa es verdadera y entra en el bloque.

Cómo pensar una condición compuesta antes de escribirla

Paso 1

Define qué decisión quieres tomar.

Paso 2

Enumera las reglas que deben cumplirse.

Paso 3

Decide si deben cumplirse todas o basta con una.

Paso 4

Usa paréntesis si la expresión puede prestarse a confusión.

Paso 5

Prueba la lógica con varios casos reales y casos límite.

Si no puedes explicar la condición en lenguaje normal antes de escribirla, probablemente aún no está bien pensada.

Código

if llueve and tengo_paraguas:
    print("Puedo salir tranquilo")
else:
    print("No puedo salir tranquilo")

Preguntas

  • ¿Qué ocurre con estos valores?
  • ¿Qué cambia si tengo_paraguas = True?
  • ¿Qué cambia si llueve = False?
Si puedes explicarlo mentalmente, es que estás entendiendo la lógica.

Tu misión

Un usuario puede beneficiarse de una promoción si:

  • es estudiante, o
  • tiene menos de 25 años
Resolver en playground

Pista

Pregúntate si la promoción exige las dos condiciones a la vez o si basta con una de ellas.

Mostrar solución 
es_estudiante = False
edad = 22

if es_estudiante or edad < 25:
    print("Tiene promoción")
else:
    print("No tiene promoción")

Errores frecuentes

Error 1

Usar or cuando en realidad se necesita and.

Error 2

Escribir comparaciones incompletas.

Error 3

Olvidar los paréntesis en expresiones complejas.

Error 4

No pensar en todos los casos posibles.

Error 5

Hacer una lógica tan larga que deja de ser legible.

Error 6

Confiar en la intuición sin evaluar paso a paso.
