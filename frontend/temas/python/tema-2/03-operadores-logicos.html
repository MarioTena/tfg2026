2.3 · Operadores lógicos: combinar condiciones de forma correcta

¿Por qué hacen falta operadores lógicos?

En problemas reales, una decisión rara vez depende de una sola regla. Normalmente necesitas combinar varias condiciones.

Ejemplos reales:

  • un usuario puede entrar si la contraseña es correcta y su cuenta está activa
  • un cliente recibe descuento si es estudiante o tiene menos de 25 años
  • se muestra un error si el pedido no está pagado o el stock es insuficiente
  • se bloquea el acceso si no ha iniciado sesión

Los tres operadores lógicos

  • and: ambas condiciones deben ser verdaderas
  • or: basta con que una sea verdadera
  • not: invierte el valor lógico

and

edad = 20
tiene_entrada = True

if edad >= 18 and tiene_entrada:
    print("Puedes pasar")
else:
    print("No puedes pasar")

Para que el resultado sea verdadero, deben cumplirse ambas condiciones. Si falla una, falla todo el bloque.

or

es_admin = False
es_supervisor = True

if es_admin or es_supervisor:
    print("Acceso permitido")
else:
    print("Acceso denegado")

Con or basta con una condición verdadera.

not

cuenta_activa = False

if not cuenta_activa:
    print("La cuenta está inactiva")

not invierte el valor lógico: convierte True en False y viceversa.

Tabla mental rápida

  • True and True → True
  • True and False → False
  • False and True → False
  • False and False → False
  • True or False → True
  • False or False → False
  • not True → False
  • not False → True

Combinar comparaciones

edad = 25

if edad >= 18 and edad <= 30:
    print("Está dentro del rango")

Esta lógica es muy común: comprobar intervalos, permisos múltiples o reglas simultáneas.

Forma corta para rangos

edad = 25

if 18 <= edad <= 30:
    print("Está dentro del rango")

Esta forma es más idiomática en Python y suele ser más legible.

⚠️ Error muy común con or

color = "rojo"

if color == "rojo" or "azul":
    print("Color permitido")

Esto está mal planteado. La segunda parte no es una comparación completa. 

color = "rojo"

if color == "rojo" or color == "azul":
    print("Color permitido")
Cada lado del or debe ser una condición completa.

Prioridad de operadores

Python no evalúa todo al azar. Hay una prioridad lógica:

  1. not
  2. and
  3. or

Si la expresión es compleja, usa paréntesis para dejar clara tu intención. 

edad = 20
es_socio = False

if edad >= 18 and (es_socio or edad > 65):
    print("Condición cumplida")

Razonar una condición paso a paso

edad = 17
tiene_permiso = True

if edad >= 16 and tiene_permiso:
    print("Puede participar")

Preguntas mentales correctas:

  • ¿edad es mayor o igual que 16? → sí
  • ¿tiene_permiso es verdadero? → sí
  • ¿ambas se cumplen? → sí
  • resultado final → entra en el bloque

Casos reales típicos

  • login correcto y cuenta activa
  • descuento por edad o por ser estudiante
  • bloqueo por falta de stock o producto inactivo
  • acceso permitido si es admin o tiene permiso especial

Errores frecuentes

  • usar or cuando en realidad se necesita and
  • escribir comparaciones incompletas
  • olvidar los paréntesis en expresiones complejas
  • no pensar en todos los casos posibles
  • hacer lógica tan larga que deja de ser legible

💭 Piensa antes de ejecutar

llueve = True
tengo_paraguas = False

if llueve and tengo_paraguas:
    print("Puedo salir tranquilo")
else:
    print("No puedo salir tranquilo")

  • ¿Qué cambia si tengo_paraguas = True?
  • ¿Qué cambia si llueve = False?

Mini-reto guiado

Un usuario puede beneficiarse de una promoción si:

  • es estudiante, o
  • tiene menos de 25 años
es_estudiante = False
edad = 22

if es_estudiante or edad < 25:
    print("Tiene promoción")
else:
    print("No tiene promoción")
