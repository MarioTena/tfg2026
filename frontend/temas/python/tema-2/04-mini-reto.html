2.4 · Mini-reto: combinar todo lo aprendido

Objetivo del reto

Ahora toca unir todo: condicionales, comparaciones, booleanos y operadores lógicos. El objetivo no es escribir muchas líneas, sino escribir lógica correcta y clara.

Enunciado

Vas a crear un pequeño sistema de acceso a una actividad. El programa tendrá estas variables:

  • edad
  • tiene_autorizacion
  • es_socio

El programa debe decidir:

  1. si puede entrar o no
  2. qué tipo de acceso tiene
  3. a qué grupo de edad pertenece

Solución guiada

edad = 16
tiene_autorizacion = True
es_socio = False

if edad >= 18 and es_socio:
    print("Acceso permitido con ventajas de socio")
elif edad >= 18:
    print("Acceso permitido")
elif edad >= 14 and tiene_autorizacion:
    print("Acceso permitido con autorización")
else:
    print("Acceso denegado")

if edad < 12:
    print("Grupo: niño")
elif edad < 18:
    print("Grupo: adolescente")
else:
    print("Grupo: adulto")

Análisis de la lógica

  • primero se evalúa el caso más específico: adulto y socio
  • después el caso general de adulto
  • después el caso del menor autorizado
  • si no se cumple nada, el acceso se deniega

Después, en un segundo bloque independiente, se clasifica la edad. Separar decisiones es una técnica útil para no mezclar reglas distintas.

Por qué esta solución es buena

  • las condiciones están ordenadas
  • los casos son mutuamente comprensibles
  • no hay lógica repetida innecesaria
  • las variables tienen nombres claros

Versión ampliada

edad = 68
tiene_autorizacion = False
es_socio = True

if edad >= 18 and es_socio:
    print("Acceso permitido con ventajas de socio")
elif edad >= 18:
    print("Acceso permitido")
elif edad >= 14 and tiene_autorizacion:
    print("Acceso permitido con autorización")
else:
    print("Acceso denegado")

if edad < 12:
    print("Grupo: niño")
elif edad < 18:
    print("Grupo: adolescente")
elif edad < 65:
    print("Grupo: adulto")
else:
    print("Grupo: senior")

Pregunta de entrevista

¿Por qué conviene separar la lógica de acceso y la lógica de clasificación en dos bloques?

Porque son decisiones distintas, con reglas distintas. Separarlas mejora claridad, mantenimiento y depuración.
