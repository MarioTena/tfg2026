Qué vas a entender hoy

Un condicional permite que el programa elija un camino u otro según una condición. Es la base de casi cualquier comportamiento inteligente en una aplicación.

Objetivo Lógica básica

Qué problema resuelve

Hasta ahora el programa ejecutaba instrucciones en línea recta. Con los condicionales, empieza a reaccionar según los datos: puede permitir acceso, mostrar un mensaje u otro o clasificar una situación automáticamente.

Acceso

Permitir o denegar entrada a un usuario según su edad o sus permisos.

Clasificación

Mostrar un resultado distinto según una nota, temperatura o puntuación.

Validación

Comprobar si un dato es correcto antes de continuar.

Decisiones reales

Aplicar reglas distintas según lo que ocurra en cada caso.

Idea clave: programar con condicionales es convertir decisiones humanas en reglas exactas para una máquina.

Qué debes vigilar

  • los dos puntos : al final de la condición
  • la indentación del bloque
  • el orden de los casos

En condicionales no basta con que la lógica “suene bien”. Tiene que estar escrita exactamente en el orden en que Python la va a evaluar.

Qué es un condicional

Definición

Un condicional permite ejecutar un bloque solo si se cumple una condición.

Idea mental correcta

El programa se hace una pregunta y actúa según la respuesta.

Respuesta posible

La condición siempre termina dando True o False.

Consecuencia

Si la condición es verdadera, entra al bloque. Si no, lo salta.

edad = 20

if edad >= 18:
    print("Puedes entrar")
Aquí la pregunta es: ¿edad es mayor o igual que 18?

Dónde aparecen los condicionales en problemas reales

Acceso

Permitir o denegar el acceso según usuario, rol o edad.

Clasificación

Mostrar mensajes distintos según nota, temperatura o nivel.

Promociones

Aplicar una regla u otra según un valor.

Validaciones

Detectar si un dato es correcto antes de seguir con el programa.

Cada vez que piensas “si pasa esto, hago esto otro”, ya estás pensando en condicionales.

Estructura básica de un if

Pieza 1

La palabra clave if.

Pieza 2

Una condición que se pueda evaluar.

Pieza 3

Los dos puntos : para abrir el bloque.

Pieza 4

Una o varias instrucciones indentadas dentro del bloque.

if condicion:
    instruccion
temperatura = 33

if temperatura > 30:
    print("Hace calor")

La sintaxis no es decorativa

En Python los bloques no se marcan con llaves. Se marcan con indentación real.

La indentación es obligatoria

Todo lo que está indentado después del if forma parte del bloque. Lo que ya no está indentado, queda fuera. 

saldo = 100

if saldo > 0:
    print("Saldo disponible")
    print("Puedes operar")

print("Fin")

Las dos líneas del centro pertenecen al condicional. La última ya no.

Error típico

edad = 20
if edad >= 18
print("Mayor de edad")

Ese código falla por dos motivos:

  • falta el :
  • el bloque no está indentado
edad = 20
if edad >= 18:
    print("Mayor de edad")

if, else y elif

if

Ejecuta algo solo si la condición es verdadera.

else

Define qué hacer cuando la condición anterior no se cumple.

elif

Permite añadir más posibilidades intermedias.

Orden

Python evalúa de arriba abajo y se detiene en la primera condición verdadera.

Dos caminos con if + else

edad = 16

if edad >= 18:
    print("Puedes votar")
else:
    print("No puedes votar")

Varias decisiones con elif

nota = 7

if nota >= 9:
    print("Sobresaliente")
elif nota >= 7:
    print("Notable")
elif nota >= 5:
    print("Aprobado")
else:
    print("Suspenso")

Dónde suele fallar la lógica

El problema más habitual no es escribir el if, sino ordenar mal los casos.

Ejemplo incorrecto

nota = 9

if nota >= 5:
    print("Aprobado")
elif nota >= 7:
    print("Notable")
elif nota >= 9:
    print("Sobresaliente")

Aunque la nota sea 9, se imprimirá Aprobado, porque la primera condición ya es verdadera y Python no sigue revisando.

Regla correcta

Cuando trabajes con rangos, normalmente conviene empezar por el caso más exigente o más restrictivo. 

nota = 8

if nota >= 9:
    print("Sobresaliente")
elif nota >= 7:
    print("Notable")
elif nota >= 5:
    print("Aprobado")
else:
    print("Suspenso")

Razonar bien los rangos

edad = 15

if edad < 12:
    print("Niño")
elif edad < 18:
    print("Adolescente")
else:
    print("Adulto")

Esto funciona porque si Python llega al segundo bloque, ya sabe que edad no era menor que 12. No hace falta repetirlo.

Una buena condición no repite información innecesaria.

Primer contacto con input()

Un programa real no siempre trabaja con variables ya escritas. Muchas veces recibe datos del usuario. 

edad = int(input("Introduce tu edad: "))

if edad >= 18:
    print("Mayor de edad")
else:
    print("Menor de edad")
Aquí el programa primero pide un dato, lo convierte a número con int() y después decide qué mensaje mostrar.

Casos frontera que debes probar

Edad mínima

Si la regla es edad >= 18, el valor 18 es el primer caso válido.

Nota mínima

Si la regla es nota >= 5, el valor 5 ya cuenta como aprobado.

Cero

En muchas clasificaciones, el 0 merece comprobarse aparte.

Cambio de bloque

Los valores frontera son los que hacen que el programa pase de una rama a otra.

edad = 18

if edad >= 18:
    print("Puede entrar")
else:
    print("No puede entrar")

Errores frecuentes del bloque

Error 1

Confundir una buena idea con una condición bien escrita.

Error 2

Olvidar los dos puntos : al final del if.

Error 3

Olvidar la indentación del bloque.

Error 4

Ordenar mal los rangos y tapar casos más específicos.

Error 5

Pensar que varios if son lo mismo que if / elif.

Error 6

No probar valores límite antes de dar el ejercicio por bueno.

Buenas prácticas

Nombres claros

Usa variables como edad, nota o tiene_permiso.

Orden lógico

Empieza por los casos más específicos cuando trabajes con rangos.

Cubre todos los casos

Comprueba que ninguna situación se quede fuera por error.

Prueba valores límite

Revisa casos como 0, 1, 17, 18 o 65 antes de darlo por bueno.
