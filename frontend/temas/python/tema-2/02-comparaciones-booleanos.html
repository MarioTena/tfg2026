Qué vas a entender hoy

Un programa decide comparando valores. Para hacerlo, necesita respuestas exactas: True o False.

Objetivo Base lógica

Qué problema resuelve

Antes de usar bien los condicionales, necesitas entender qué tipo de preguntas puede responder Python. Las comparaciones son esas preguntas, y los booleanos son sus respuestas.

Edad

Comprobar si alguien es mayor o igual que 18.

Acceso

Ver si un usuario coincide con el esperado.

Precio

Detectar si una cantidad supera un límite.

Validación

Saber si dos datos son iguales o distintos.

Idea clave: una comparación responde una pregunta exacta y el resultado siempre es booleano.

Comparaciones y booleanos: la base de todo

Comparación

Es una expresión que enfrenta dos valores para responder una pregunta exacta.

Booleano

Es un tipo de dato que solo puede valer True o False.

Resultado

Toda comparación termina produciendo un booleano.

Uso real

Es la base de validaciones, filtros, clasificaciones y decisiones.

print(5 > 3)
print(10 == 10)
print(7 < 2)

aprobado = True
bloqueado = False
print(type(aprobado))

Dónde aparecen las comparaciones en problemas reales

Acceso

Comprobar si un usuario o contraseña coincide con lo esperado.

Filtros

Decidir si un precio, una edad o una nota supera un umbral.

Validación

Detectar si un dato es correcto antes de continuar.

Clasificación

Separar casos según igualdades, diferencias o rangos.

Cada vez que haces una pregunta del tipo “¿es igual?”, “¿es mayor?” o “¿coincide?”, estás comparando.

El error más importante del bloque

Esta confusión rompe muchísimos ejercicios al principio.

= no es ==

  • = significa asignar
  • == significa comparar
edad = 18          # asignación
print(edad == 18)  # comparación

La primera línea guarda un valor. La segunda pregunta si ese valor es igual a 18.

Guardar el resultado de una comparación

edad = 17
es_mayor = edad >= 18
print(es_mayor)

Aquí es_mayor vale False.

Cómo comparar bien

==

igual que

!=

distinto de

> / <

mayor que / menor que

>= / <=

mayor o igual / menor o igual

Con números

precio = 50
print(precio > 100)
print(precio <= 50)
print(precio == 50)
print(precio != 30)

Con texto

usuario = "ana"
print(usuario == "ana")
print(usuario == "Ana")
print(usuario != "admin")
print("5" == 5)        # False
print("Ana" == "ana")  # False
print(10 == 10.0)      # True
"ana" y "Ana" no son el mismo valor. "5" y 5 tampoco.

Cómo se usan dentro de un if

nota = 6

if nota >= 5:
    print("Has aprobado")
else:
    print("Has suspendido")

El if no evalúa directamente el número: evalúa el booleano que produce la comparación.

Comparaciones encadenadas

edad = 20
print(18 <= edad <= 30)

Esto equivale a edad >= 18 and edad <= 30, pero es más limpio.

Más allá de comparar

Truthiness

Python interpreta algunos valores como verdaderos o falsos aunque no sean comparaciones.

0

Suele interpretarse como falso.

Cadena vacía

"" suele interpretarse como falsa.

Valor con contenido

Un número distinto de 0 o una cadena con texto suelen ser verdaderos.

print(bool(0))
print(bool(1))
print(bool(""))
print(bool("Hola"))
activo = True
print(activo == True)
print(activo == False)

Cómo pensar una comparación antes de escribirla

Paso 1

Identifica los dos valores que vas a enfrentar.

Paso 2

Decide qué relación quieres comprobar: igual, distinto, mayor o menor.

Paso 3

Comprueba si el tipo de dato tiene sentido.

Paso 4

Piensa en el caso límite o frontera.

Paso 5

Pregúntate si el resultado esperado debería ser True o False.

Muchos errores con comparaciones no vienen del operador, sino de comparar cosas que no deberías comparar así.

Piensa antes de ejecutar

Antes de probar el código, intenta predecir los resultados.

Errores frecuentes

Error 1

Confundir asignación con comparación.

Error 2

Comparar texto sin tener en cuenta mayúsculas y minúsculas.

Error 3

Olvidar que toda comparación devuelve un booleano.

Error 4

No probar valores límite.

Error 5

Comparar valores que parecen iguales, pero no tienen el mismo tipo.

Error 6

Hacer comparaciones innecesariamente rebuscadas.

Buenas prácticas

Pregunta exacta

Antes de escribir, formula mentalmente la pregunta que quieres responder.

Tipos correctos

Comprueba si estás comparando números con números y texto con texto.

Casos límite

Revisa siempre valores frontera como 0, 1, 18 o 100.

Código claro

Haz comparaciones simples y separadas cuando eso facilite entender el resultado.
