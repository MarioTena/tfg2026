Qué vas a entender hoy

Un programa decide comparando valores. Para hacerlo, necesita respuestas exactas: True o False.

Objetivo Base lógica

Qué problema resuelve

Antes de usar bien los condicionales, necesitas entender qué tipo de preguntas puede responder Python. Las comparaciones son esas preguntas, y los booleanos son sus respuestas.

Edad

Comprobar si alguien es mayor o igual que 18.

Acceso

Ver si un usuario coincide con el esperado.

Precio

Detectar si una cantidad supera un límite.

Validación

Saber si dos datos son iguales o distintos.

Idea clave: una comparación responde una pregunta exacta y el resultado siempre es booleano.

Comparaciones y booleanos: la base de todo

Comparación

Es una expresión que enfrenta dos valores para responder una pregunta exacta.

Booleano

Es un tipo de dato que solo puede valer True o False.

Resultado

Toda comparación termina produciendo un booleano.

Uso real

Es la base de validaciones, filtros, clasificaciones y decisiones.

print(5 > 3)
print(10 == 10)
print(7 < 2)

aprobado = True
bloqueado = False
print(type(aprobado))

Dónde aparecen las comparaciones en problemas reales

Acceso

Comprobar si un usuario coincide con el esperado.

Filtros

Decidir si un precio, una edad o una nota supera un umbral.

Validación

Detectar si un dato es correcto antes de continuar.

Clasificación

Separar casos según igualdades, diferencias o rangos.

Cada vez que haces una pregunta del tipo “¿es igual?”, “¿es mayor?” o “¿coincide?”, estás comparando.

El error más importante del bloque

Esta confusión rompe muchísimos ejercicios al principio.

= no es ==

  • = significa asignar
  • == significa comparar
edad = 18          # asignación
print(edad == 18)  # comparación

La primera línea guarda un valor. La segunda pregunta si ese valor es igual a 18.

Guardar el resultado de una comparación

edad = 17
es_mayor = edad >= 18
print(es_mayor)

Aquí es_mayor vale False.

Cómo comparar bien

==

igual que

!=

distinto de

> / <

mayor que / menor que

>= / <=

mayor o igual / menor o igual

Con números

precio = 50
print(precio > 100)
print(precio <= 50)
print(precio == 50)
print(precio != 30)

Con texto

usuario = "ana"
print(usuario == "ana")
print(usuario == "Ana")
print(usuario != "admin")
print("5" == 5)        # False
print("Ana" == "ana")  # False
print(10 == 10.0)      # True
"ana" y "Ana" no son el mismo valor. "5" y 5 tampoco.

Cómo se usan dentro de un if

nota = 6

if nota >= 5:
    print("Has aprobado")
else:
    print("Has suspendido")

El if no evalúa directamente el número: evalúa el booleano que produce la comparación.

Variables booleanas dentro de un if

es_mayor = True

if es_mayor:
    print("Puede pasar")
else:
    print("No puede pasar")

Un if no necesita siempre una comparación escrita en ese momento. También puede evaluar directamente una variable booleana. 

cuenta_activa = False

if cuenta_activa:
    print("La cuenta está activa")
else:
    print("La cuenta está inactiva")

Comparaciones encadenadas

edad = 20
print(18 <= edad <= 30)

Esta forma es muy útil para comprobar rangos sencillos con claridad. 

temperatura = 25

if 20 <= temperatura <= 30:
    print("Temperatura templada")

Casos frontera: donde más se falla

> frente a >=

No significan lo mismo. Uno excluye el valor límite y el otro lo incluye.

El 18

Si la regla es edad >= 18, 18 ya entra como válido.

El 5

Si la regla es nota >= 5, 5 ya es aprobado.

El 0

Cuando clasificas números, el 0 suele merecer una comprobación específica.

nota = 5

if nota >= 5:
    print("Aprobado")
else:
    print("Suspenso")

Primer contacto con input()

nota = int(input("Introduce tu nota: "))

if nota >= 5:
    print("Aprobado")
else:
    print("Suspenso")

Aquí el programa recibe el dato del usuario, lo convierte con int() y después decide.

Errores que debes detectar rápido

Aprender comparaciones también consiste en reconocer fallos comunes.

Error 1

edad = 18
if edad = 18:
    print("Coincide")

Está mal porque = asigna; no compara.

Error 2

usuario = "Ana"
print(usuario == "ana")

Da False porque las mayúsculas y minúsculas importan.

Buenas prácticas

Tipos correctos

Casos límite

Revisa siempre valores frontera como 0, 5, 18 o 100.

Código claro

Haz comparaciones simples cuando eso facilite entender el resultado.
