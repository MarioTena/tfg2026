Qué entrenas aquí

Aquí dejas de usar condiciones aisladas y empiezas a construir lógica completa y ordenada.

if / elif / else Booleanos Lógica

Objetivo

Estos retos te ayudan a decidir bien qué condición va primero, cómo combinar reglas y cómo separar decisiones distintas.

Idea clave: en un condicional, el orden influye directamente en el resultado.

Cómo usar esta pantalla

  • lee el reto
  • identifica qué se está decidiendo
  • piensa qué caso es más específico
  • pruébalo en el playground con distintos valores
No basta con que “entre por algún sitio”. La lógica debe ser clara, ordenada y predecible.

Misiones rápidas

Mini-reto 1 · Mayor o menor de edad

Decide qué mensaje mostrar según la edad.

edad = 17

if edad >= 18:
    print("Mayor de edad")
else:
    print("Menor de edad")

Mini-reto 2 · Nota aprobada

Comprueba si una nota supera el mínimo.

nota = 6

if nota >= 5:
    print("Aprobado")
else:
    print("Suspenso")

Mini-reto 3 · Acceso administrador

Usa una comparación con texto.

usuario = "admin"

if usuario == "admin":
    print("Acceso total")
else:
    print("Acceso limitado")

Mini-reto 4 · Descuento por edad o estudio

Combina dos condiciones con or.

edad = 22
es_estudiante = False

if edad < 25 or es_estudiante:
    print("Tiene descuento")
else:
    print("No tiene descuento")

Mini-reto 5 · Acceso seguro

Combina dos requisitos con and.

password_correcta = True
cuenta_activa = True

if password_correcta and cuenta_activa:
    print("Acceso permitido")
else:
    print("Acceso denegado")

Mini-reto 6 · Clasificación de edad

Ordena bien varias condiciones con elif.

edad = 16

if edad < 12:
    print("Niño")
elif edad < 18:
    print("Adolescente")
else:
    print("Adulto")

Cómo pensar este tipo de ejercicios

Antes de escribir condiciones, define bien la decisión que quieres tomar.

Paso a paso

  1. identifica qué pregunta responde el programa
  2. ordena de lo más específico a lo más general
  3. si hay varias reglas, sepáralas mentalmente
  4. usa and y or solo cuando hagan falta
  5. prueba siempre casos límite

Regla útil

Si una condición general va antes que una específica, puede impedir que la específica llegue a evaluarse.
