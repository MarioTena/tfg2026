Qué vas a entender hoy

Leer un archivo significa abrirlo correctamente y recuperar su contenido completo o línea a línea según lo que necesites hacer.

Objetivo Leer contenido

Qué problema resuelve

Guardar datos en un archivo no sirve de mucho si luego no sabes recuperarlos. Leer archivos te permite reutilizar información guardada por ti o por otros programas.

Dónde aparece leer archivos en problemas reales

Configuración

Cargar valores que el programa necesita al arrancar.

Listas de datos

Leer registros, tareas o nombres guardados previamente.

Procesamiento

Recuperar información para analizarla o recorrerla línea a línea.

Persistencia útil

Volver a usar contenido que no vive ya en variables temporales.

Leer archivos aparece cuando el programa necesita recuperar información guardada fuera de la ejecución actual.

Lectura completa

archivo = open("datos.txt", "r")
contenido = archivo.read()
archivo.close()
print(contenido)

Esto lee todo el contenido del archivo de una vez.

Leer línea a línea

archivo = open("datos.txt", "r")
for linea in archivo:
    print(linea)
archivo.close()

Es muy útil cuando el archivo tiene varias líneas y quieres procesarlas una a una.

Usar with

with open("datos.txt", "r") as archivo:
    contenido = archivo.read()
    print(contenido)
with es la forma recomendada porque cierra el archivo automáticamente.

Por qué with no es solo “más bonito”

Más seguro

Evita olvidos al cerrar el archivo.

Más limpio

Reduce pasos repetitivos en el código.

Más claro

Deja ver mejor dónde empieza y termina el trabajo con el archivo.

Mejor hábito

Es el patrón que conviene interiorizar desde el principio.

Lectura completa o línea a línea

Completa

Útil si el archivo es pequeño y quieres todo su contenido de golpe.

Línea a línea

Útil si el archivo tiene estructura por líneas o quieres procesarlo poco a poco.

Completa

Encaja cuando solo quieres mostrar o guardar el texto entero.

Línea a línea

Encaja cuando vas a contar, filtrar o limpiar líneas individualmente.

Leer no es lo mismo que procesar

Leer un archivo solo trae el contenido a tu programa. Después puedes:

  • mostrarlo
  • contar líneas
  • buscar palabras
  • limpiar espacios o saltos
  • quedarte solo con parte de la información
Primero recuperas el contenido. Después decides qué hacer con él.

Las líneas suelen traer salto de línea

with open("datos.txt", "r") as archivo:
    for linea in archivo:
        print(linea)

Muchas veces cada línea ya incluye su propio salto de línea al final. Por eso, al imprimir directamente, el resultado puede verse con espacios o líneas extrañas.

Limpiar con strip()

with open("datos.txt", "r") as archivo:
    for linea in archivo:
        print(linea.strip())

Esto ayuda a mostrar cada línea de forma más limpia cuando no quieres espacios o saltos sobrantes.

Error típico: archivo no encontrado

Si la ruta o el nombre del archivo es incorrecto, el programa falla porque no puede localizarlo.

Casos frontera útiles

Archivo vacío

read() devuelve texto vacío.

Una sola línea

Puedes leerlo completo o recorrerlo, según tu objetivo.

Muchas líneas

Suele tener más sentido procesar poco a poco.

Líneas “sucias”

Puede que necesites strip() para mostrarlas como quieres.

Cómo pensar una lectura antes de escribirla

Paso 1

Comprueba que el archivo existe y el nombre es correcto.

Paso 2

Decide si quieres todo el contenido o línea a línea.

Paso 3

Usa with para evitar olvidos al cerrar.

Paso 4

Piensa qué harás con el contenido una vez leído.

Piensa antes de ejecutar

Antes de escribir código, piensa qué forma de lectura encaja mejor con el contenido del archivo.

Situaciones

  • leer una nota corta completa
  • procesar un archivo con muchas líneas
  • evitar olvidarte de cerrar el archivo

Preguntas
