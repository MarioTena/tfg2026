Qué es un archivo

Aprende por qué los archivos permiten guardar información fuera del programa y reutilizarla mucho después de que termine la ejecución.

Qué vas a entender hoy

Un archivo permite guardar datos fuera del programa para que no desaparezcan cuando termina la ejecución.

Objetivo Guardar fuera

Qué problema resuelve

Hasta ahora los datos vivían en variables temporales. Cuando el programa se cerraba, toda esa información desaparecía. Los archivos permiten conservarla para usarla después.

Idea clave: una variable dura mientras el programa vive; un archivo puede durar mucho más.

Qué debes vigilar

  • un archivo guarda datos fuera de la memoria del programa
  • para trabajar con él debes abrirlo
  • después debes leer o escribir
  • al terminar, conviene cerrarlo correctamente
Piensa así: trabajar con archivos es hacer que los datos sobrevivan al final del programa.

Los archivos aparecen cuando necesitas que la información sobreviva más allá de una ejecución concreta del programa.

Variable frente a archivo

Variable

Vive en memoria mientras el programa se está ejecutando.

Archivo

Permanece guardado fuera del programa y puede abrirse más tarde.

Variable

Desaparece al cerrar el programa.

Archivo

Puede seguir existiendo mañana, otro día o incluso después de reiniciar el equipo.

El archivo es una fuente externa de datos

Un archivo no vive “dentro” de tu código. Está fuera, en una ruta del sistema. Por eso trabajar con archivos también implica aceptar que algo externo puede fallar: el archivo puede no existir, tener otro nombre o estar en otra ubicación.

Esto hace que trabajar con archivos sea distinto a trabajar solo con variables.

La idea general

Para trabajar con archivos, normalmente sigues este patrón:

  1. abrir el archivo
  2. leer o escribir en él
  3. cerrarlo correctamente
Es un flujo muy repetido: abrir → trabajar → cerrar.

Función open()

archivo = open("datos.txt", "r")

Esto abre el archivo en modo lectura.

El primer dato es el nombre o ruta del archivo. El segundo indica el modo en que quieres abrirlo.

Modos habituales

"r"

Leer el contenido.

"w"

Escribir sobrescribiendo lo anterior.

"a"

Añadir contenido al final.

Elección correcta

Elegir mal el modo puede hacer que pierdas datos.

Nombre y ruta del archivo

Para abrir un archivo necesitas indicar su nombre o ruta. Si el programa no lo encuentra exactamente donde espera, la operación fallará. 

open("notas.txt", "r")
open("carpeta/notas.txt", "r")

No hace falta profundizar mucho todavía, pero sí entender que el nombre y la ubicación importan.

Cómo pensar el modo correcto antes de abrir un archivo

Paso 1

Pregúntate si solo quieres leer.

Paso 2

Pregúntate si quieres crear un contenido nuevo.

Paso 3

Pregúntate si debes conservar lo anterior.

Paso 4

Elige entre r, w o a según esa intención.

Piensa antes de ejecutar

Aquí la clave no es la sintaxis, sino entender qué quieres hacer con el archivo antes de abrirlo.
