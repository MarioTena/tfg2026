Qué vas a entender hoy

Un archivo permite guardar datos fuera del programa para que no desaparezcan cuando termina la ejecución.

Objetivo Guardar fuera

Qué problema resuelve

Hasta ahora los datos vivían en variables temporales. Cuando el programa se cerraba, toda esa información desaparecía. Los archivos permiten conservarla para usarla después.

Resultados

Conservar la salida de un programa para reutilizarla.

Idea clave: una variable dura mientras el programa vive; un archivo puede durar mucho más.

Qué debes vigilar

  • un archivo guarda datos fuera de la memoria del programa
  • para trabajar con él debes abrirlo
  • después debes leer o escribir
  • al terminar, conviene cerrarlo correctamente
Piensa así:
trabajar con archivos es hacer que los datos sobrevivan al final del programa.

La idea general

Para trabajar con archivos, normalmente sigues este patrón:

  1. abrir el archivo
  2. leer o escribir en él
  3. cerrarlo correctamente
Es un flujo muy repetido: abrir → trabajar → cerrar.

Función open()

archivo = open("datos.txt", "r")

Esto abre el archivo en modo lectura.

El primer dato es el nombre o ruta del archivo. El segundo indica el modo en que quieres abrirlo.

Modos habituales

"r"

Leer el contenido.

"w"

Escribir sobrescribiendo lo anterior.

"a"

Añadir contenido al final.

Elección correcta

Elegir mal el modo puede hacer que pierdas datos.

Cómo pensar el modo correcto antes de abrir un archivo

Paso 1

Pregúntate si solo quieres leer.

Paso 2

Pregúntate si quieres crear un contenido nuevo.

Paso 3

Pregúntate si debes conservar lo anterior.

Paso 4

Elige entre r, w o a según esa intención.

Piensa antes de ejecutar

Aquí la clave no es la sintaxis, sino entender qué quieres hacer con el archivo antes de abrirlo.

Reto Muy fácil

Tu misión

Identifica qué modo usarías en cada caso:

  • leer una lista de tareas
  • crear un informe nuevo
  • añadir una línea al historial

Solución razonada

  • leer una lista de tareas → "r"
  • crear un informe nuevo → "w"
  • añadir una línea al historial → "a"
La clave está en pensar si quieres leer, reemplazar o conservar lo que ya había.

Errores frecuentes

Error 1

Pensar que los datos de variables duran para siempre.

Error 2

No distinguir entre leer y escribir.

Error 3

Elegir un modo incorrecto al abrir el archivo.

Error 4

No entender que el archivo vive fuera del programa.
