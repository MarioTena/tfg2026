Combina for, while, break y continue para resolver pequeños problemas con iteraciones.

Qué entrenas aquí

Aquí ya no se trata solo de entender la sintaxis. Se trata de repetir con lógica y controlar cuándo continúa o termina una iteración.

for while Control

Objetivo

Vas a empezar a usar bucles para resolver problemas pequeños pero muy típicos: contar, acumular, filtrar y validar.

Idea clave: en un bucle, lo importante no es repetir, sino entender qué cambia en cada vuelta.

Cómo usar esta pantalla

  • lee el reto
  • sigue mentalmente las iteraciones
  • comprueba qué variable cambia y cuándo termina
  • pruébalo en el playground y modifica casos
Antes de ejecutar, intenta predecir cuántas iteraciones habrá y qué salida producirá el programa.

Misiones rápidas

Mini-reto 1 · Contar hacia atrás

Muestra una cuenta atrás del 5 al 1.

for numero in range(5, 0, -1):
    print(numero)

Mini-reto 2 · Sumar del 1 al 10

Usa un acumulador para obtener el total.

suma = 0

for numero in range(1, 11):
    suma = suma + numero

print(suma)

Mini-reto 3 · Mostrar solo impares

Usa continue para saltar casos que no interesan.

for numero in range(1, 11):
    if numero % 2 == 0:
        continue
    print(numero)

Mini-reto 4 · Detener al encontrar un valor

Usa break cuando ya no tenga sentido seguir.

for letra in "programacion":
    if letra == "g":
        break
    print(letra)

Mini-reto 5 · Validación con while

Repite hasta que el usuario escriba la palabra correcta.

texto = ""

while texto != "python":
    texto = input("Escribe python: ")

print("Correcto")

Mini-reto 6 · Acumulación con while

Haz que el contador y la suma evolucionen correctamente.

contador = 1
suma = 0

while contador <= 5:
    suma = suma + contador
    contador = contador + 1

print(suma)

Cómo pensar este tipo de ejercicios

Antes de escribir, identifica bien el patrón de repetición.

Paso a paso

  1. identifica qué se repite
  2. decide si conviene for o while
  3. piensa qué variable cambia
  4. define cuándo termina
  5. si hace falta, añade break o continue

Regla útil

Si sabes cuántas repeticiones o qué secuencia recorres, suele encajar for. Si dependes de una condición abierta, suele encajar while.
