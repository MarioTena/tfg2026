break y continue

Aprende a controlar mejor el flujo de un bucle: detenerlo antes o saltar iteraciones concretas.

Tema 3 · Bucles
Nivel inicial · 12-15 min · Control de flujo
break continue
Empezar misión Abrir playground

Qué vas a entender hoy

A veces un bucle no debe completar todas sus iteraciones. Necesitas poder pararlo o saltarte una vuelta concreta.

Objetivo Control

Qué problema resuelve

Hay situaciones en las que seguir iterando ya no tiene sentido, o donde una iteración concreta no te interesa. Ahí entran break y continue.

Búsqueda

Cuando ya has encontrado lo que buscabas, puedes detener el bucle.

Filtrado

Puedes saltarte elementos que no te interesan.

Validación

Un programa puede terminar una repetición antes de tiempo si ya tiene una respuesta suficiente.

Claridad

Bien usados, ayudan a expresar mejor la intención del algoritmo.

Idea clave: break detiene el bucle entero; continue solo salta la iteración actual.

Qué debes vigilar

  • si quieres parar el bucle o solo saltar una vuelta
  • en qué condición exacta se ejecuta break o continue
  • cómo cambia la salida del programa al introducirlos
  • que la lógica siga siendo fácil de leer
No uses break o continue por costumbre. Úsalos cuando mejoren la lógica y la claridad.

Qué hace break

for letra in "python":
    if letra == "h":
        break
    print(letra)

Cuando Python llega a break, el bucle termina por completo.

Qué hace continue

for numero in range(1, 6):
    if numero == 3:
        continue
    print(numero)

Cuando Python llega a continue, no ejecuta el resto de esa iteración y pasa a la siguiente.

Diferencia mental rápida

break

“No tiene sentido seguir. Salgo del bucle.”

continue

“Esta vuelta no me interesa. Paso a la siguiente.”

Efecto

Uno corta el proceso; el otro lo mantiene pero salta una iteración.

Uso real

Buscar, filtrar, validar o evitar trabajo innecesario.

Dónde aparecen break y continue en problemas reales

Búsqueda

Dejar de recorrer cuando ya has encontrado el dato deseado.

Filtrado

Ignorar elementos que no cumplen una condición.

Validación

Salir cuando se detecta la primera respuesta correcta.

Optimización

Evitar vueltas que ya no aportan nada al resultado.

Ejemplo con break en while

while True:
    texto = input("Escribe salir para terminar: ")

    if texto == "salir":
        break

print("Programa terminado")

Este patrón se usa mucho cuando la salida depende de una orden del usuario.

Ejemplo con continue para filtrar

for numero in range(1, 11):
    if numero % 2 == 0:
        continue
    print(numero)

Aquí se saltan los números pares y solo se imprimen los impares.

Qué cambia en la salida

for numero in range(1, 6):
    if numero == 4:
        break
    print(numero)

Se imprimen 1, 2 y 3. El 4 ya no se imprime porque el bucle se corta justo antes. 

for numero in range(1, 6):
    if numero == 4:
        continue
    print(numero)

Se imprimen 1, 2, 3 y 5. El 4 se omite, pero el bucle sigue.

Errores frecuentes

Error 1

Confundir break con continue.

Error 2

No pensar si el corte ocurre antes o después del print.

Error 3

Usarlos sin entender qué efecto tienen en la salida final.

Error 4

Complicar un bucle sencillo con demasiados saltos.
