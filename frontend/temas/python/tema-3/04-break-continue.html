3.4 · break y continue: controlar el flujo dentro de un bucle

¿Qué problema resuelven break y continue?

A veces no basta con repetir. Necesitas modificar el comportamiento del bucle desde dentro:

  • detenerlo antes de tiempo
  • saltar una iteración concreta

Para eso sirven break y continue.

break: terminar el bucle inmediatamente

for numero in range(1, 10):
    if numero == 5:
        break
    print(numero)

Cuando numero vale 5, el bucle se detiene por completo. Ya no hay más iteraciones.

Cuándo usar break

  • cuando encuentras lo que buscabas
  • cuando ocurre una condición de parada
  • cuando no tiene sentido seguir iterando

continue: saltar solo la iteración actual

for numero in range(1, 6):
    if numero == 3:
        continue
    print(numero)

Aquí no se imprime el 3, pero el bucle sigue con el 4 y el 5.

Diferencia exacta entre ambos

  • break → corta todo el bucle
  • continue → salta solo una vuelta y sigue con la siguiente

Ejemplo realista con búsqueda

for letra in "python":
    if letra == "h":
        break
    print(letra)

Se imprimen p, y, t y luego el bucle termina.

Ejemplo realista con filtrado

for numero in range(1, 8):
    if numero % 2 == 0:
        continue
    print(numero)

Aquí saltas los pares y solo muestras impares.

⚠️ Error típico: usar break cuando querías continue

Si quieres saltarte un valor, no debes romper el bucle entero. 

for numero in range(1, 6):
    if numero == 3:
        break
    print(numero)

Esto no “omite el 3”. Hace algo más radical: termina completamente el bucle.

Con while también existen

contador = 0

while True:
    contador = contador + 1
    if contador == 4:
        break
    print(contador)

Este patrón se usa mucho cuando la condición de salida se decide desde dentro del bucle.

💭 Piensa antes de ejecutar

for numero in range(1, 6):
    if numero == 2:
        continue
    if numero == 5:
        break
    print(numero)
  • ¿qué números se imprimen?
  • ¿por qué no aparece el 2?
  • ¿por qué no aparece el 5?

Buenas prácticas

  • usa break solo cuando la salida anticipada tenga sentido real
  • usa continue para filtrar casos concretos
  • no abuses de ellos si hacen el flujo difícil de leer
  • explica la intención con variables y condiciones claras

Mini-reto guiado

Recorre los números del 1 al 10 e imprime todos menos el 5. 

for numero in range(1, 11):
    if numero == 5:
        continue
    print(numero)

Reto de entrevista

¿Qué diferencia conceptual hay entre break y continue?

Respuesta sólida: break termina el bucle completo; continue solo omite la iteración actual y permite seguir con las siguientes.
