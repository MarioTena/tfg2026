Qué entrenas aquí

Este reto te obliga a trabajar con datos numéricos que deben validarse antes de analizarlos.

Integración Validación Análisis

Objetivo

Vas a pedir notas al usuario hasta que escriba fin. Después, el programa deberá calcular estadísticas útiles.

Idea clave: aquí no basta con guardar números; además debes validar que tengan sentido y luego analizarlos.

Qué conceptos estás combinando

  • input
  • while
  • listas
  • validación de rango
  • condicionales
  • cálculo de media
  • conteo de aprobados y suspensos

Enunciado

Crea un programa que pida notas al usuario hasta que escriba fin. Cada nota debe ser un número entre 0 y 10.

Al terminar, el programa debe mostrar:

  • cuántas notas se han introducido
  • la media
  • la nota más alta
  • la nota más baja
  • cuántos aprobados hay
  • cuántos suspensos hay

Requisitos mínimos

  • debes usar una lista para guardar las notas
  • debes validar que cada nota esté entre 0 y 10
  • debes usar un bucle while
  • debes parar cuando el usuario escriba fin
  • si no hay notas, debes indicarlo claramente

Diseño recomendado

Paso 1

Pide notas una a una hasta que el usuario escriba fin.

Paso 2

Comprueba primero si la entrada es fin antes de convertirla a número.

Paso 3

Valida que cada nota esté entre 0 y 10 antes de guardarla en la lista.

Paso 4

Cuando termine la entrada, calcula media, máximo, mínimo y número de aprobados y suspensos.

Fases del reto

Separar entrada, validación y análisis te ayudará a no mezclar demasiadas cosas a la vez.

Fase 1

Haz que el programa pida notas hasta que el usuario escriba fin.

Fase 2

Comprueba primero si la entrada es fin antes de convertirla.

Fase 3

Valida que la nota esté entre 0 y 10 antes de guardarla.

Fase 4

Guarda las notas válidas en una lista.

Fase 5

Cuando termine la entrada, calcula media, máximo, mínimo, aprobados y suspensos.

Fase 6

Mejora los mensajes y comprueba también el caso en que no se haya introducido ninguna nota.

Pistas opcionales

Pista 1

Convierte la entrada a número solo después de comprobar que no es fin.

Pista 2

Valida con una condición tipo 0 <= nota <= 10.

Pista 3

La media es suma total dividida entre cantidad de notas.

Pista 4

Una nota aprobada suele ser mayor o igual que 5.

Errores frecuentes

Error 1

Convertir a número antes de comprobar si el usuario ha escrito fin.

Error 2

Guardar notas fuera del rango permitido.

Error 3

Calcular la media sin comprobar antes si hay notas suficientes.

Error 4

Mezclar validación y análisis sin separar bien las fases del programa.

Versión ampliada

  • mostrar también cuántas notas son excelentes, notables y suspensas
  • usar funciones separadas para validar y analizar
  • ordenar las notas antes de mostrarlas

Qué deberías haber consolidado al terminar

Este reto te obliga a tratar datos numéricos con más rigor que en ejercicios sueltos.

Competencias que consolidan

  • validar números dentro de un rango
  • guardar notas en una lista
  • calcular media, máximo y mínimo
  • contar aprobados y suspensos
  • gestionar correctamente el caso sin datos

Cierre didáctico

Aquí ya no basta con hacer cálculos: primero debes decidir qué datos aceptas y solo después analizarlos con sentido.
