Qué entrenas aquí

Este reto te obliga a usar juntos varios conceptos que ya conoces para recoger datos, guardarlos y analizarlos al final.

Integración Razonamiento Práctica real

Objetivo

Vas a crear un programa que pida números al usuario hasta que escriba fin. Después, el programa deberá analizar todos los números introducidos.

Idea clave: aquí ya no basta con leer un número y mostrar algo. Tienes que guardar varios datos, recorrerlos y extraer información útil.

Qué conceptos estás combinando

  • input
  • while
  • listas
  • conversión con int()
  • condicionales
  • contadores
  • máximo y mínimo

Enunciado

Crea un programa que pida números al usuario uno por uno. El programa debe seguir pidiendo números hasta que el usuario escriba fin.

Cuando termine la entrada, el programa debe mostrar:

  • cuántos números ha introducido
  • cuántos son pares
  • cuántos son impares
  • cuál es el mayor
  • cuál es el menor

Requisitos mínimos

  • debes usar un bucle while
  • debes usar una lista para guardar los números
  • debes parar cuando el usuario escriba fin
  • debes convertir a entero los números válidos
  • debes mostrar el análisis final al terminar
  • si no se ha introducido ningún número, debes indicarlo claramente
No intentes resolver el análisis al mismo tiempo que lees. Primero guarda bien los datos y luego piensa cómo recorrerlos.

Diseño recomendado

Paso 1

Crea una lista vacía para guardar los números.

Paso 2

Pide datos al usuario dentro de un while.

Paso 3

Si escribe fin, termina el bucle.

Paso 4

Si escribe un número, conviértelo y guárdalo en la lista.

Paso 5

Cuando termine, recorre la lista para analizar sus datos.

Paso 6

Muestra el resumen final por pantalla.

Fases del reto

Avanzar por fases te ayudará a no mezclar demasiadas cosas a la vez.

Fase 1

Haz que el programa pida datos hasta que el usuario escriba fin.

Fase 2

Guarda cada número válido en una lista.

Fase 3

Muestra cuántos números se han guardado.

Fase 4

Cuenta cuántos son pares y cuántos son impares.

Fase 5

Calcula el mayor y el menor.

Fase 6

Mejora mensajes y comprueba el caso donde no se introducen números.

Pistas opcionales

Pista 1

Usa una lista vacía, por ejemplo numeros = [].

Pista 2

Compara la entrada con "fin" antes de convertirla.

Pista 3

Puedes usar append() para guardar cada número en la lista.

Pista 4

Para saber si un número es par, usa numero % 2 == 0.

Pista 5

Para el mayor y el menor, puedes usar max() y min() si ya los has visto, o razonarlo manualmente.

Pista 6

Antes de analizar, comprueba si la lista está vacía.

Errores frecuentes

Error 1

Convertir a número antes de comprobar si el usuario ha escrito fin.

Error 2

Intentar analizar los datos mientras todavía se están recogiendo, en vez de separar entrada y análisis.

Error 3

Calcular mayor o menor sin pensar qué pasa si no se ha introducido ningún número.

Error 4

Confundir contar números pares e impares con contar cuántos datos se han introducido en total.

Versión ampliada

Si terminas la versión principal, puedes añadir mejoras como estas:

  • calcular también la media
  • contar positivos, negativos y ceros
  • evitar entradas no numéricas usando try / except si ya has visto errores
  • mostrar la lista completa al final
