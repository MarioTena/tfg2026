Qué entrenas aquí

Este reto sirve como puente entre retos y proyectos: te obliga a organizar un programa pequeño con menú y varias acciones.

Menú Funciones Organización

Objetivo

Vas a crear un programa con menú donde el usuario pueda introducir datos, ver un resumen y salir.

Qué conceptos estás combinando

  • while
  • if / elif / else
  • funciones
  • listas
  • validación de opción
  • menús

Enunciado

Crea un programa con menú que permita:

  • añadir un dato a una lista
  • mostrar los datos guardados
  • mostrar un resumen simple
  • salir

El programa debe seguir funcionando hasta que el usuario elija salir.

Requisitos mínimos

  • el menú debe repetirse hasta que el usuario elija salir
  • debes validar las opciones introducidas
  • debes guardar los datos en una lista
  • debes mostrar los datos actuales cuando se solicite
  • debes incluir al menos un resumen simple sobre los datos guardados

Diseño recomendado

Paso 1

Crea una lista vacía y un bucle principal para mantener el menú activo.

Paso 2

Implementa primero las opciones mínimas: añadir dato y salir.

Paso 3

Después añade mostrar datos y el resumen final.

Paso 4

Cuando funcione, separa cada acción en una función distinta para organizar mejor el programa.

Fases del reto

Un menú se construye mejor por capas: primero el flujo, luego las acciones y al final la organización.

Fase 1

Crea la lista vacía y un bucle principal que muestre el menú repetidamente.

Fase 2

Haz funcionar primero la opción de añadir dato.

Fase 3

Añade la opción de mostrar los datos guardados.

Fase 4

Implementa un resumen simple sobre la lista.

Fase 5

Valida bien las opciones del menú para evitar flujos rotos.

Fase 6

Cuando todo funcione, separa el comportamiento en funciones para que el programa quede más limpio.

Pistas opcionales

Pista 1

Usa un while True para mantener el menú activo.

Pista 2

Crea una función para mostrar el menú.

Pista 3

Valida las opciones con if / elif / else.

Pista 4

Haz primero una versión mínima con solo dos opciones.

Errores frecuentes

Error 1

Intentar construir todas las opciones del menú a la vez sin validar primero el flujo básico.

Error 2

No validar la opción elegida y dejar el programa en estados confusos.

Error 3

Mezclar el menú, las acciones y el resumen en un único bloque muy difícil de leer.

Error 4

No contemplar el caso en que aún no haya datos guardados.

Versión ampliada

  • permitir borrar un dato
  • mostrar el número total de elementos
  • evitar entradas vacías
  • separar cada acción en una función distinta

Qué deberías haber consolidado al terminar

Este reto es una preparación clara para programas más largos y para tus proyectos integradores.

Competencias que consolidan

  • mantener un flujo activo con menú
  • validar opciones del usuario
  • guardar y mostrar datos de forma consistente
  • organizar el programa en funciones
  • separar interfaz, almacenamiento y análisis

Cierre didáctico

Aquí empiezas a diseñar programas con estructura: ya no resuelves una sola acción, sino un pequeño sistema que debe mantenerse coherente mientras el usuario interactúa con él.
