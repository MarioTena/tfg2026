Qué entrenas aquí

En este bloque no solo escribes funciones. Aprendes a leerlas, entender qué reciben, qué hacen y qué producen realmente.

Lectura Depuración Práctica guiada

Objetivo

Estos retos te ayudan a dejar de ver una función como “una cosa con def” y empezar a verla como una pieza con contrato: qué hace, qué recibe y qué devuelve.

Idea clave: una función correcta no solo funciona. También deja claro qué tarea resuelve y qué resultado produce.

Cómo usar esta pantalla

  • lee el código antes de ejecutarlo
  • identifica definición, parámetros, llamada y resultado
  • pregúntate si la función debería imprimir o devolver
  • detecta errores de llamada, diseño o ámbito
Si no puedes explicar qué entra en la función y qué sale de ella, todavía no controlas del todo su diseño.

Reto 1 · Definir no es llamar

Código

def saludar():
    print("Hola")

Qué debes pensar

¿Ese código imprime algo ya o solo define la función? ¿Qué faltaría para ejecutarla?

Reto 2 · Parámetro y argumento

Código

def saludar(nombre):
    print("Hola", nombre)

saludar("Ana")

Pregunta

¿Qué parte es el parámetro y qué parte es el argumento? ¿Qué valor toma nombre dentro de la función?

Reto 3 · Error de llamada

Código incorrecto

def saludar(nombre):
    print("Hola", nombre)

saludar()

Pregunta

¿Por qué falla exactamente? ¿Qué le falta a la llamada para respetar el contrato de la función?

No basta con decir “da error”. Debes identificar qué esperaba la función y qué recibió realmente.

Reto 4 · print o return

Código

def doble(numero):
    print(numero * 2)

resultado = doble(4)
print(resultado)

Pregunta

¿Qué se imprime realmente? ¿Por qué aparece None al final?

Este es uno de los errores más importantes del tema: confundir mostrar un resultado con devolverlo.

Reto 5 · return termina la función

Código

def clasificar(numero):
    if numero > 0:
        return "positivo"
    print("Esto no siempre se ejecuta")

Qué entrenas

Piensa qué pasa cuando la función entra en el return. ¿Sigue leyendo líneas o se detiene?

Reto 6 · Variable local

Código incorrecto

def crear_mensaje():
    mensaje = "Hola"

print(mensaje)

Pregunta

¿Por qué falla? ¿Dónde existe realmente mensaje?

Reto 7 · Diseño de función limpia

Código

def hacer_cosa(x):
    print("Resultado:", x * 1.21)

Pregunta

¿Qué mejorarías en este diseño? Piensa en el nombre y en si conviene imprimir o devolver.

Reto 8 · Reutilización real

Código

def cuadrado(numero):
    return numero * numero

a = cuadrado(3)
b = cuadrado(5)
print(a + b)

Prueba obligatoria

Explica qué devuelve la función, qué se guarda en a y b, y qué imprime la última línea.

Método rápido para comprobar una función

Qué hacer antes de dar una función por bien diseñada.

Paso a paso

  1. identifica qué tarea exacta resuelve
  2. comprueba qué datos necesita
  3. decide si debe imprimir o devolver
  4. revisa si su nombre comunica bien la intención
  5. haz al menos dos llamadas con datos distintos

Regla útil

Si una función funciona pero no sabes explicar qué recibe y qué produce, todavía no la has entendido del todo.
