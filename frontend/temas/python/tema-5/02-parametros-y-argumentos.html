Parámetros y argumentos

Aprende a hacer funciones flexibles para que puedan trabajar con distintos datos sin cambiar su lógica.

Qué vas a entender hoy

Una función fija sirve, pero una función realmente útil debe poder trabajar con datos distintos.

Objetivo Funciones flexibles

Qué problema resuelve

Si una función siempre hace lo mismo con los mismos datos, sirve poco. Los parámetros permiten reutilizar la misma lógica con valores diferentes.

Saludos

Mostrar mensajes a distintas personas sin escribir una función por cada una.

Cálculos

Usar la misma función con números diferentes.

Reutilización

Evitar cambiar el código interno cada vez que cambian los datos.

Claridad

Separar la lógica de la función de los datos concretos que recibe.

Idea clave: los parámetros hacen que una función sea reutilizable con valores distintos.

Qué debes vigilar

  • no confundir parámetro con argumento
  • respetar el número de argumentos esperados
  • usar nombres claros
  • entender que la lógica de la función no cambia, solo cambian los datos
Piensa así:
una función con parámetros es como una plantilla: la estructura se mantiene, pero los valores pueden cambiar.

Dónde aparecen parámetros y argumentos en problemas reales

Mensajes

Enviar el mismo formato de saludo a diferentes usuarios.

Cálculos

Aplicar una operación a distintos importes, notas o cantidades.

Procesos

Reutilizar una función con entradas diferentes según el contexto.

Automatización

Evitar crear una función nueva para cada caso concreto.

¿Por qué hacen falta parámetros?

Una función fija sirve, pero una función realmente útil debe poder trabajar con datos distintos. Para eso existen los parámetros. 

def saludar(nombre):
    print("Hola", nombre)

saludar("Ana")
saludar("Luis")

Diferencia entre parámetro y argumento

Parámetro

La variable que aparece en la definición de la función.

Argumento

El valor real que pasas al llamar a la función.

Relación

El argumento se asigna al parámetro cuando llamas a la función.

Idea práctica

Uno define el hueco; el otro lo rellena.

def saludar(nombre):
    print("Hola", nombre)

saludar("Ana")

Aquí nombre es el parámetro y "Ana" es el argumento.

Más de un parámetro

def presentar(nombre, edad):
    print("Me llamo", nombre)
    print("Tengo", edad, "años")

presentar("Ana", 30)

El orden importa si llamas a la función por posición.

Error típico: número incorrecto de argumentos

def saludar(nombre):
    print("Hola", nombre)

saludar()

Esto da error porque la función espera un argumento y no ha recibido ninguno.

Los nombres internos no tienen por qué coincidir con las variables externas

def saludar(persona):
    print("Hola", persona)

nombre_usuario = "Marta"
saludar(nombre_usuario)

El argumento se asigna al parámetro interno al llamar a la función.

Funciones con cálculo interno

def mostrar_doble(numero):
    print(numero * 2)

mostrar_doble(5)
mostrar_doble(12)

La función recibe un valor y actúa con él.

Parámetros con nombres claros

Igual que con las variables, los parámetros deben describir bien lo que representan. 

def calcular_precio_con_iva(precio):
    print(precio * 1.21)

Argumentos literales y variables

def saludar(nombre):
    print("Hola", nombre)

saludar("Carlos")

usuario = "Lucía"
saludar(usuario)

Puedes pasar directamente un valor o una variable que lo contenga.

Cómo pensar una función con parámetros antes de escribirla

Paso 1

Identifica qué parte de la tarea siempre será igual.

Paso 2

Detecta qué datos van a cambiar entre una llamada y otra.

Paso 3

Convierte esos datos cambiantes en parámetros.

Paso 4

Pon nombres a los parámetros que indiquen bien qué representan.

Si una función se repite cambiando solo algunos datos, probablemente necesita parámetros.

Piensa antes de ejecutar

Antes de probar, separa la lógica fija de los valores que cambian.

Código

def multiplicar_por_tres(numero):
    print(numero * 3)

multiplicar_por_tres(4)
multiplicar_por_tres(10)

Preguntas

  • ¿qué se imprime en cada llamada?
  • ¿la lógica de la función cambia o solo cambian los datos?
Pista: la función es la misma en ambos casos.

Mini-reto guiado

Ahora toca hacer una función reutilizable con un dato que cambia.

Reto Muy fácil

Tu misión

Crea una función que reciba una ciudad y muestre un mensaje de bienvenida.

Resolver en playground

Pista

Aquí la estructura del mensaje no cambia, solo cambia el valor de la ciudad.

Mostrar solución 
def bienvenida_ciudad(ciudad):
    print("Bienvenido a", ciudad)

bienvenida_ciudad("Madrid")
bienvenida_ciudad("Bilbao")

Errores frecuentes

Error 1

Confundir parámetro con argumento.

Error 2

Pasar menos o más argumentos de los necesarios.

Error 3

Usar nombres demasiado ambiguos.

Error 4

Pensar que una función “recuerda” automáticamente valores externos.

Buenas prácticas

Parámetros claros

Usa nombres que representen bien el dato recibido.

Lógica separada

No mezcles la idea de la función con valores concretos del exterior.

Comprueba llamadas

Revisa siempre si estás pasando el número correcto de argumentos.

Piensa en reutilización

Diseña la función para que sirva con varios datos, no solo con un caso.
