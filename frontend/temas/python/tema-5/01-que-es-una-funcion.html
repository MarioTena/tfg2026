Qué vas a entender hoy

Una función permite encapsular una tarea concreta en un bloque reutilizable con nombre propio.

Objetivo Reutilización

Qué problema resuelve

A medida que los programas crecen, empiezan a aparecer bloques de código que se repiten. Copiar y pegar puede parecer rápido al principio, pero empeora el código.

Duplicación

Copiar lógica varias veces hace el código más difícil de mantener.

Errores

Si cambias una parte y olvidas otra, aparecen inconsistencias.

Lectura

El programa se vuelve más largo y menos claro.

Organización

Las funciones ayudan a dividir un problema grande en piezas pequeñas.

Idea clave: una función te permite definir una tarea una vez y usarla tantas veces como necesites.

Qué debes vigilar

  • definir no es lo mismo que ejecutar
  • los paréntesis son obligatorios
  • la indentación sigue siendo esencial
  • el nombre debe describir bien la tarea
Piensa así:
una función es como una herramienta con nombre: la creas una vez y la usas cuando la necesitas.

Dónde aparecen las funciones en problemas reales

Validaciones

Comprobar varias veces una misma regla sin copiar código.

Cálculos

Repetir operaciones como descuentos, impuestos o medias.

Interfaces

Mostrar menús, mensajes o bloques reutilizables.

Programas grandes

Separar el código por tareas para que sea más fácil de entender.

En software real, casi todo está organizado en funciones.

Qué es una función

Una función es un bloque de código que realiza una tarea concreta. Se define una vez y puede ejecutarse tantas veces como sea necesario. 

def saludar():
    print("Hola")

Aquí hemos definido una función llamada saludar. Pero definirla no significa ejecutarla.

Definir no es lo mismo que llamar

def saludar():
    print("Hola")

saludar()

La primera parte crea la función. La segunda la llama para que se ejecute.

Sintaxis básica

def nombre_funcion():
    bloque

def

Indica que vas a definir una función.

Nombre

Debe describir la tarea que realiza.

Paréntesis

Son obligatorios aunque no haya parámetros todavía.

Bloque

El código interior va indentado, igual que en otras estructuras.

Por qué las funciones son tan importantes

Reutilizan código

Evitan repetir la misma lógica muchas veces.

Dividen problemas

Permiten separar tareas complejas en partes más pequeñas.

Mejoran la lectura

Un buen nombre hace que el código se entienda mejor.

Facilitan pruebas

Es más fácil revisar una función pequeña que un bloque enorme.

Ejemplo de problema sin función

print("Hola Ana")
print("Bienvenida al sistema")

print("Hola Luis")
print("Bienvenido al sistema")

Aquí hay repetición. La estructura es la misma, aunque cambien algunos datos.

Idea de función como herramienta reutilizable

Piensa en una función como una pequeña máquina: le das un nombre, sabes qué hace, y la usas cuando la necesitas.

Funciones incorporadas y funciones propias

Ya has usado funciones aunque quizá no lo notaras: 

print("Hola")
len([1, 2, 3])

print() y len() son funciones incorporadas de Python. Ahora empiezas a crear las tuyas.

Cómo pensar una función antes de escribirla

Paso 1

Identifica una tarea que se repite o que merece tener nombre propio.

Paso 2

Define claramente qué hace esa tarea y qué no hace.

Paso 3

Asigna un nombre que describa una acción.

Paso 4

Decide cuándo se llamará dentro del programa.

Muchas veces una buena función nace de detectar una repetición o una tarea demasiado larga.

Piensa antes de ejecutar

Antes de probar el código, intenta seguir el flujo del programa.

Código

def mostrar_mensaje():
    print("Estoy dentro de la función")

print("Antes")
mostrar_mensaje()
print("Después")

Preguntas

  • ¿qué se imprime primero?
  • ¿qué línea provoca la ejecución de la función?
Pista: definir una función no la ejecuta todavía.

Mini-reto guiado

Ahora toca encapsular una tarea sencilla en una función propia.

Reto Muy fácil

Tu misión

Crea una función llamada mostrar_bienvenida que imprima dos líneas de saludo.

Resolver en playground

Pista

Primero define la función con def y después llámala con su nombre y paréntesis.

Mostrar solución 
def mostrar_bienvenida():
    print("Bienvenido")
    print("Vamos a aprender funciones")

mostrar_bienvenida()

Errores frecuentes

Error 1

Olvidar los paréntesis al llamar a la función.

Error 2

Pensar que definir una función la ejecuta automáticamente.

Error 3

Poner mal la indentación del bloque.

Error 4

Elegir nombres demasiado ambiguos.

Buenas prácticas

Nombres con acción

Usa nombres como saludar, calcular_total o mostrar_menu.

Una responsabilidad

Haz que cada función tenga una tarea clara y concreta.

Tamaño razonable

Evita funciones demasiado largas al empezar.

Piensa antes de crear

No conviertas cualquier línea en función: hazlo cuando aporte claridad o reutilización.
