Qué entrenas aquí

En este proyecto das un salto importante: tus datos ya no viven solo mientras el programa está abierto. Ahora los guardas en archivo y los recuperas después.

Persistencia Validación Programa real

Objetivo

Vas a crear un programa que guarde registros en un archivo, permita leerlos de nuevo, añadir información nueva y controlar errores básicos.

Idea clave: aquí el programa ya maneja información que sobrevive al cierre, algo muy cercano al comportamiento real de muchas aplicaciones.

Qué conceptos estás combinando

  • archivos
  • lectura y escritura
  • funciones
  • validación
  • try / except
  • raise
  • strings
  • listas

Situación

Imagina que quieres guardar registros sencillos, por ejemplo nombres, tareas, productos o cualquier dato básico. El programa debe poder añadir nuevos registros y recuperar los que ya existían.

Leer

Mostrar lo que ya estaba guardado.

Añadir

Guardar nuevas líneas sin borrar lo anterior.

Validar

Evitar guardar entradas vacías o inválidas.

Gestionar errores

Responder mejor si hay un problema con el archivo o con la entrada.

Requisitos mínimos

  • debes usar un archivo de texto para guardar información
  • debes usar funciones para organizar el programa
  • debes mostrar un menú dentro de un bucle while
  • debes permitir ver registros guardados
  • debes permitir añadir un nuevo registro
  • debes validar que el registro no esté vacío
  • debes manejar errores básicos relacionados con lectura o escritura
  • debes permitir salir
Este proyecto te obliga a combinar almacenamiento real, validación y control de errores en un mismo flujo.

Estructura recomendada del archivo

Para este nivel, la opción más sencilla es guardar un registro por línea. Cada línea puede contener un texto simple, por ejemplo una tarea, un nombre o un dato básico.

Diseño recomendado

Función 1

Mostrar el menú principal.

Función 2

Leer y mostrar los registros del archivo.

Función 3

Añadir un nuevo registro al archivo.

Función 4

Validar que el dato introducido sea correcto.

Función 5

Gestionar errores con mensajes útiles.

Bucle principal

Controlar el menú y las acciones del usuario.

Flujo mental del proyecto

  1. preguntar qué quiere hacer el usuario
  2. si quiere ver datos, leer el archivo
  3. si quiere añadir datos, validar la entrada
  4. guardar el nuevo registro usando el modo correcto
  5. si algo falla, mostrar un mensaje comprensible
Aquí el gran aprendizaje no es solo usar archivos, sino decidir bien cuándo leer, cuándo añadir y cómo reaccionar si algo sale mal.

Fases del proyecto

Resuélvelo por partes para que el programa vaya creciendo con orden.

Fase 1

Crea el menú y la opción de salir.

Fase 2

Implementa la lectura del archivo y muestra las líneas guardadas.

Fase 3

Implementa la opción de añadir un nuevo registro.

Fase 4

Valida que la entrada no esté vacía antes de guardarla.

Fase 5

Captura errores básicos del archivo con try / except.

Fase 6

Refuerza mensajes y mejora la claridad del programa.

Pistas opcionales

Pista 1

Usa with open(..., "r") para leer.

Pista 2

Usa with open(..., "a") para añadir sin borrar lo anterior.

Pista 3

Recuerda usar \n si quieres guardar cada registro en una línea distinta.

Pista 4

Usa .strip() para limpiar lo leído o validar que no esté vacío.

Pista 5

Captura FileNotFoundError si el archivo no existe al intentar leer.

Pista 6

Puedes usar raise ValueError(...) si la entrada no cumple la regla que quieres imponer.

Errores frecuentes

Error 1

Usar el modo de apertura incorrecto y borrar sin querer lo que ya había en el archivo.

Error 2

No validar que el registro tenga contenido antes de guardarlo.

Error 3

Leer o escribir sin gestionar qué ocurre si el archivo no existe o falla la operación.

Error 4

Mezclar persistencia, validación y navegación del menú sin separar responsabilidades.

Versión ampliada

Si terminas la versión principal, puedes añadir mejoras como estas:

  • guardar solo registros válidos y rechazar los inválidos
  • mostrar cuántos registros hay en total
  • crear un archivo nuevo con solo los registros válidos
  • permitir buscar un registro por texto
  • guardar la fecha o la categoría junto al registro

Qué deberías haber conseguido al terminar

Este proyecto te acerca mucho más a la programación útil de verdad.

Competencias que consolidan

  • leer y escribir datos persistentes
  • elegir correctamente entre leer y añadir
  • validar entradas antes de guardar
  • gestionar errores previsibles
  • organizar todo eso con funciones y menú

Cierre didáctico
