Qué entrenas aquí

En este proyecto ya no practicas una sola instrucción. Combinas varias piezas del lenguaje para construir un programa completo con menú, funciones y estado.

Integración Consola Programa completo

Objetivo

Vas a crear un gestor de tareas en consola. El usuario podrá interactuar con el programa mediante un menú y realizar acciones sobre una lista de tareas.

Idea clave: este proyecto te obliga a usar juntas varias cosas que hasta ahora habías practicado por separado.

Qué conceptos estás combinando

  • input y print
  • if / elif / else
  • while
  • listas
  • funciones
  • recorrido con for y enumerate()
  • validación básica de entradas

Situación

Imagina que quieres crear un programa sencillo para organizar tareas del día a día. El usuario debe poder añadir tareas, verlas, eliminarlas y marcar cuáles ya están hechas.

Ver tareas

Consultar qué tareas hay actualmente guardadas.

Añadir

Guardar una nueva tarea escrita por el usuario.

Eliminar

Borrar una tarea por su número.

Completar

Marcar una tarea como realizada de forma visible.

Requisitos mínimos

  • debes usar una lista para guardar tareas
  • debes mostrar un menú dentro de un bucle while
  • debes usar funciones para organizar el código
  • el programa debe permitir ver tareas
  • el programa debe permitir añadir una tarea
  • el programa debe permitir eliminar una tarea
  • el programa debe permitir marcar una tarea como completada
  • el programa debe permitir salir
  • no debes aceptar tareas vacías
  • si la opción del menú es incorrecta, debes avisarlo
No hace falta que tu programa sea perfecto. Sí hace falta que sea claro, usable y que combine correctamente los conceptos.

Cómo puedes representarlo

Para este nivel, la forma más sencilla es guardar las tareas como textos dentro de una lista. Cuando una tarea se complete, puedes añadir "✅ " al principio del texto. 

tareas = [
    "Estudiar Python",
    "✅ Comprar pan",
    "Hacer ejercicio"
]
No es la solución más avanzada, pero sí una muy buena solución para este momento de aprendizaje.

Bucle principal

Controlar el programa hasta que el usuario decida salir.

Fases del proyecto

Te conviene construirlo por partes, no intentar hacerlo entero de golpe.

Fase 1

Crea el menú y haz que el programa pueda salir correctamente.

Fase 2

Añade la opción de guardar tareas nuevas en la lista.

Fase 3

Muestra las tareas numeradas con enumerate().

Fase 4

Permite eliminar una tarea por su número.

Fase 5

Permite marcar una tarea como completada.

Fase 6

Mejora la validación de opciones y de entradas.

Pistas opcionales

Pista 1

Usa un while True para mantener el menú activo.

Pista 2

Usa break cuando el usuario elija salir.

Pista 3

Para mostrar tareas numeradas, usa enumerate(tareas, start=1).

Pista 4

Antes de eliminar o completar, comprueba que el número esté dentro del rango válido.

Pista 5

Usa .strip() para detectar tareas vacías.

Pista 6

Para marcar completada, puedes comprobar si el texto empieza por "✅ ".

Versión ampliada

Si terminas la versión principal, puedes añadir mejoras opcionales como estas:

  • no permitir completar una tarea ya completada
  • mostrar cuántas tareas hay en total
  • mostrar cuántas están completadas
  • permitir editar una tarea
  • mostrar tareas pendientes y completadas por separado

Qué deberías haber conseguido al terminar

Este proyecto marca un salto importante respecto a los ejercicios sueltos.

Competencias que consolidan

  • mantener datos en una lista
  • usar funciones para dividir un problema
  • controlar un programa mediante un menú
  • modificar información ya guardada
  • validar entradas del usuario
